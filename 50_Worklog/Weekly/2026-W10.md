# 2026年第9周周报

## 1️⃣ 完成软限位问题分析并确认解决策略

本周针对设备触发软限位后无法正常移动的问题进行了远程调试与开发沟通，明确了软限位控制逻辑：

- 触发软限位被系统视为**故障场景**
- 在执行绝对位置移动前，系统会判断目标位置是否越界
- 若越界则禁止绝对定位移动
- 因此 **只能通过反向点动方式脱离软限位区域**

同时确认当前系统已经支持读取软限位相关参数，可在上位机或逻辑层提前进行位置合法性校验，避免进入非法区域。

**成果：**

- 明确软限位触发后的控制策略
- 确认点动脱离为唯一合法恢复方式
- 为后续上位机安全控制逻辑提供依据

## 3️⃣ 梳理 servoV6 控制系统架构设计（Domain / UseCase）

在项目推进过程中，对工业控制软件架构进行了进一步思考，重点梳理：

### Domain 层

负责描述设备业务模型，例如：

- 电机使能
- 点动
- 绝对定位
- 停止
- 位置读取

该层只表达业务逻辑，不依赖具体技术实现（如 Modbus、寄存器、通信协议）。

### UseCase 层

负责实现**具体业务流程**，例如：

- 电机点动流程
- 联动控制流程
- 设备运行流程

通过 Domain 接口组织业务逻辑。

### ViewModel 层

负责：

- 管理 UI 状态
- 调用 UseCase
- 将设备状态转换为 UI 可展示数据

**成果：**

- 明确工业控制软件分层结构
- 区分业务模型与业务流程
- 为后续 Qt + PLC 控制系统架构奠定基础
