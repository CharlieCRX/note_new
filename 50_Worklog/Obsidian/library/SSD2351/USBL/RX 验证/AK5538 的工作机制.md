## 阶段零：上电初始态（物理层待机）
- **状态**：芯片刚上电（AVDD/TVDD 稳定），但 **PDN 引脚（Pin 28）为低电平**。
- **内部动作**：内部 LDO（低压差线性稳压器）关闭，所有模拟电路（ADC 调制器）和数字电路（滤波器、接口）均处于**断电复位（Power-Down）**状态。此时，不管你怎么发 I2C 命令，芯片都“装死”，I2C 寄存器虽可写入但无效，SDTO 数据引脚输出为 0。

---

## 阶段一：硬件解复位（配置生效的总开关）
- **你的操作**：`fpga -w 0x5 0x1F`（通过 FPGA 拉高 AK5538 的 PDN 引脚）。
- **芯片内部反应**：
  1. **LDO 启动**：内部 1.8V 数字核心供电建立。
  2. **参考电压建立**：内部带隙基准源启动，VREFH 引脚建立稳定的 3.3V 参考电压。
  3. **寄存器解冻**：此时，I2C 写入的值才被真正加载到内部工作寄存器中。你的 `Control_1 = 0x05` 和 `Power_Man2 = 0x01` 在这一刻才正式“刻入”芯片逻辑。
- **关键点**：这步不执行，后面的全是白费。

---

## 阶段二：静态寄存器配置（设定“游戏规则”）
- **你的操作**：`i2cset` 写入 `0x01`、`0x02`。
- **芯片内部反应**（按寄存器功能划分）：

  | 寄存器                | 你写的值               | 芯片内部实际逻辑动作                                         |
  | :-------------------- | :--------------------- | :----------------------------------------------------------- |
  | **Power_Man2 (0x01)** | 先 `0x00` 后 `0x01`    | **Bit0 (RSTN) 从 0→1**：释放内部数字滤波器的复位。**关键**：这一步让数字抽取滤波器（Decimation Filter）做好接收数据的准备。 |
  | **Control_1 (0x02)**  | `0x05`                 | **Bit7-4 (CKS)=0000**：设定时钟分频器为 `128fs`。告诉内部 PLL：“待会儿进来的 MCLK 是采样率的 128 倍”。<br>**Bit3-2 (DIF)=01**：设定串行输出协议为 **I2S 格式**（左对齐，MSB 优先）。配置输出移位寄存器如何打包数据。 |
  | **Power_Man1 (0x00)** | （未显式写，默认全 1） | **Bit0-7 开启**：导通模拟输入缓冲器（Input Buffer）的偏置电流。此时模拟前端已通电，但因无时钟，处于静默等待。 |

---

## 阶段三：供给主时钟（启动“心脏起搏器”）
- **你的操作**：`fpga -w 0x2 6`（向 AK5538 的 **24 脚 MCLK** 注入 16.384MHz 方波）。
- **芯片内部反应**（这是最具技术含量的瞬间）：
  1. **时钟检测**：MCLK 引脚检测到上升沿跳变，内部时钟缓冲器（Buffer）开启。
  2. **PLL（锁相环）锁定**：芯片根据 `CKS=0000` 的设定，将 16.384MHz 进行分频。因为 16.384MHz ÷ 128 = **128kHz**，芯片内部生成了 **LRCK（帧时钟）** 和 **BCLK（位时钟）** 的逻辑基础时序。
  3. **调制器时钟生成**：PLL 进一步将 MCLK 倍频或分频，生成 ADC 核心所需的 **极高频率调制时钟（如 6.144MHz 或更高）**，这是 Δ-Σ 调制器工作的“发动机转速”。

---

## 阶段四：内部初始化与稳定（等待“热身”）
- **你的操作**：（没有额外命令，纯硬件自动执行，耗时约 4.5ms @ 128kHz）。
- **芯片内部反应**（依据数据手册 Page 55）：
  1. **计数周期**：从 MCLK 稳定起，内部逻辑开始计数 **578 个 LRCK 周期**。
  2. **静音输出**：在此期间，芯片 **SDTO（数据输出引脚）被硬件强制拉为 0**（静音），防止在未稳定时输出乱码。
  3. **调制器稳定**：Δ-Σ 调制器内部的电容阵列完成电荷建立，数字滤波器完成流水线填充。

---

## 阶段五：模拟 → 数字转换（物理世界的“翻译官”）
- **外部条件**：AIN1P/AIN1N 差分引脚输入 ±1.85V 的模拟音频信号。
- **芯片内部反应（信号链核心）**：
  1. **差分采样**：模拟输入缓冲器将差分信号送入 **Δ-Σ 调制器**。
  2. **过采样（Oversampling）**：调制器以调制时钟（如 6.144MHz）对模拟信号进行高速采样，将模拟电压转换为 **1 比特高速数据流（PDM 流）**。
  3. **抽取滤波（Decimation）**：这一串 1 比特流进入数字抽取滤波器。滤波器将高速低精度的数据，**降低采样率（抽取）并提高位宽**，最终计算出 32-bit 的 PCM 音频数据。
  4. **高通滤波器（HPF）**：根据 `Control_1` 的 Bit1=0（关闭），数据直接绕过 HPF，保留直流分量。

---

## 阶段六：数字域打包与串行输出（“快递”发货）
- **芯片内部反应**：
  1. **多路复用**：计算好的 8 通道 32-bit PCM 数据被送入 **输出数据移位寄存器**。
  2. **协议封装**：移位寄存器根据 `Control_1` 设定的 **I2S 协议**（DIF=01），在 BCLK 的下降沿将数据按 **MSB 优先** 的顺序对齐到 LRCK 的左右声道周期中。
  3. **引脚驱动**：在 BCLK 驱动下，数据从 **SDTO1 ~ SDTO4** 引脚（取决于通道数）串行移出，最终通过 PCB 走线流向你的 FPGA。

---

## 总结：你的命令与芯片步骤的一一映射

| 你的 Shell 命令         | AK5538 内部发生的物理/逻辑事件                               |
| :---------------------- | :----------------------------------------------------------- |
| `fpga -w 0x5 0x1F`      | **点火**：PDN 拉高，LDO 和偏置电路启动，寄存器解锁。         |
| `i2cset 0x01 0x00/0x01` | **挂挡**：释放数字滤波器复位，准备接收数据通路。             |
| `i2cset 0x02 0x05`      | **定方向盘**：设定采样率倍频 128x 和输出协议 I2S。           |
| `fpga -w 0x2 6`         | **踩油门**：注入 16.384MHz MCLK，内部 PLL 锁定，生成调制时钟。 |
| （硬件自动完成）        | **怠速热车**：578 个周期初始化，输出静音。                   |
| （模拟信号输入）        | **发动机做功**：Δ-Σ 调制 + 抽取滤波，生成 PCM 数据。         |
| （BCLK/LRCK 驱动）      | **传动输出**：将数据按 I2S 时序从 SDTO 引脚送到 FPGA。       |

**一句话概括**：寄存器定义“怎么跑”，PDN 解复位允许“跑”，MCLK 提供“跑的力气”，最后内部状态机自动完成从模拟电压到串行数字比特流的物理转换。你现在只要在 FPGA 端把 SDTO 引脚的 I2S 数据抓下来，就能看到真实的音频波形了。🎯
