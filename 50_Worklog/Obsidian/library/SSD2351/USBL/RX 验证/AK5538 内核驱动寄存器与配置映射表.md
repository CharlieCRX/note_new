## 2. 详细寄存器位域定义

### 2.1 寄存器 0x00：电源管理 1 (PW8 - PW1)
- **功能**：独立控制 8 个通道（AIN1 ~ AIN8）的 ADC 电源开关。
- **位定义**：`PWn` (n=1~8)
    - `0`：对应通道 **关闭（Power Down）**，输出静音（全 0）。
    - `1`：对应通道 **开启（正常操作）**。
- **默认值**：`0xFF`（所有通道默认开启）。

### 2.2 寄存器 0x01：电源管理 2 (MONO2, MONO1, RSTN)
- **位定义**：
    - **RSTN (位 0)**：内部时序复位（非寄存器复位）。
        - `0`：复位内部数字滤波器和时序。
        - `1`：正常工作（默认）。
    - **MONO2 (位 3), MONO1 (位 2)**：通道求和模式（提升动态范围）。

| MONO2 | MONO1 | 模式名称 | 描述 |
| :--- | :--- | :--- | :--- |
| 0 | 0 | **非求和模式 (Normal)** | 8 通道独立输出（默认） |
| 0 | 1 | **8-to-2 模式** | 将 AIN1-4 求和输出到 Slot 1，AIN5-8 求和输出到 Slot 2 |
| 1 | 0 | **8-to-4 模式** | 将 (1,2), (3,4), (5,6), (7,8) 两两求和输出到 4 个 Slot |
| 1 | 1 | **8-to-1 模式** | 将全部 8 个通道求和输出到 Slot 1 |

### 2.3 寄存器 0x02：控制 1 (CKS3-0, DIF1-0, HPFE)
- **CKS3-0（位 7-4）**：**时钟模式选择**（详见第 3 节采样率映射表）。
- **DIF1-0（位 3-2）**：**音频数据格式选择**（I2S / MSB 对齐 / 位宽）。

| DIF1 | DIF0 | 格式描述 |
| :--- | :--- | :--- |
| 0 | 0 | 24-bit MSB 对齐（左对齐），标准模式 |
| 0 | 1 | **24-bit I2S 兼容格式（常用）** |
| 1 | 0 | 32-bit MSB 对齐 |
| 1 | 1 | **32-bit I2S 兼容格式** |

- **HPFE（位 0）**：高通滤波器（HPF）使能。`1` 使能，`0` 旁路（默认）。

### 2.4 寄存器 0x03：控制 2 (TDM1-0)
- **TDM1-0（位 7-6）**：**TDM 模式选择**。

| TDM1 | TDM0 | 模式 | BICK 频率要求 |
| :--- | :--- | :--- | :--- |
| 0 | 0 | **Normal（非 TDM）** | 48fs ~ 128fs |
| 0 | 1 | **TDM128** | 128fs |
| 1 | 0 | **TDM256** | 256fs |
| 1 | 1 | **TDM512** | 512fs |

### 2.5 寄存器 0x04：控制 3 (DP, SD, SLOW)
- **DP（位 7）**：输出模式选择。`0` = PCM 模式，`1` = DSD 模式。
- **SD（位 1）, SLOW（位 0）**：数字滤波器类型选择（仅 PCM 模式）。

| SD (位1) | SLOW (位0) | 滤波器类型 |
| :--- | :--- | :--- |
| 0 | 0 | **Sharp Roll-off Filter（陡降）**（推荐，默认） |
| 0 | 1 | Slow Roll-off Filter（缓降） |
| 1 | 0 | Short Delay Sharp Roll-off |
| 1 | 1 | Short Delay Slow Roll-off |

## 3. 采样率与 MCLK 配置表（核心）

**关键点**：`CKS3-0` 位决定了 `MCLK` 与采样率 (`fs`) 的倍率关系，以及对应的 `fs` 有效范围。

根据数据手册表 5（Table 5）提取的常用配置如下：

| 采样率 (fs) | 速度模式 | MCLK 倍率 | MCLK 频率示例 (匹配 USBL 平台) | **CKS3-0 值 (寄存器 0x02 高 4 位)** |
| :--- | :--- | :--- | :--- | :--- |
| **128 kHz** | **Quad Speed** | **128fs** | 16.384 MHz | **0x0 (0000)** |
| 192 kHz | Quad Speed | 128fs 或 192fs | 24.576 MHz 或 36.864 MHz | 0x0 或 0x1 |
| 48 kHz | Normal Speed | 256fs | 12.288 MHz | 0x4 (0100) |
| 48 kHz | Normal Speed | 512fs | 24.576 MHz | 0xC (1100) |
| 96 kHz | Double Speed | 256fs | 24.576 MHz | 0x5 (0101) |
| 32 kHz | Normal Speed | 256fs | 8.192 MHz | 0x4 |
| 384 kHz | Oct Speed | 64fs | 24.576 MHz | 0x8 (1000) |
| 768 kHz | Hex Speed | 32fs 或 48fs | 24.576 MHz | 0x9 (1001) |

> **针对你当前测试的 128kHz**：
> 请务必设置 **`CKS3-0 = 0x0`**，选择 **128fs** 模式。此时 CPU（SSD2351）应输出 16.384 MHz 的 MCLK 和 128kHz 的 LRCK。

## 4. 驱动初始化（上电）流程建议

结合数据手册第 55 页（Power Up/Down Sequence），在驱动 `probe` 或 `hw_params` 时应遵循以下顺序：

1.  **硬件解复位（外部引脚）**：
    - 确保 `PDN` 引脚拉高（如果硬件已由 FPGA 或 GPIO 控制）。
2.  **配置寄存器（建议序列）**：
    - 写 `0x01` 寄存器，设置 `RSTN = 0`（复位内部时序）。
    - 写 `0x02` 寄存器，配置 `CKS`（采样率）和 `DIF`（I2S 格式）。**务必先配时钟，再释放复位**。
    - 写 `0x03` 寄存器，配置 `TDM` 模式（若使用 TDM）。
    - 写 `0x04` 寄存器，配置 `DP`（PCM 模式）和滤波器。
    - 最后，写 `0x01` 寄存器，设置 `RSTN = 1`（释放内部复位）。
3.  **时钟供应**：
    - 上述配置完成后，再使能外部 MCLK 时钟（或确保时钟稳定存在）。
4.  **等待初始化完成**：
    - 根据数据手册，从 `PDN` 拉高到数据有效，需要等待 `578 ~ 583 / fs` 秒（约 4.5ms @ 128kHz）。建议驱动中增加 `msleep(5)` 或检查状态寄存器。
