## 架构分块

riscv-v架构分成了两个模块：

- ~~基础命令模块：为教学或者加速器等目标提供基本的指令~~
- ~~可选的拓展模块：支持通用的软件设计目标~~

### "基础命令模块" 不够准确

这里强调的是：

> **基础整数 ISA（Base Integer ISA）**

它并不是"为了教学而设计"。它真正的定位是：

> **一个完整、最小、可以独立运行的 ISA。**

教学和加速器只是它**能够应用的两个场景（use cases）**。

正确的是：

**RISC-V 将 ISA 分为两个部分：**

- **基础整数 ISA（Base ISA）**：提供最小但完整的一组基础整数指令，可以独立实现，也适合作为教学平台或定制硬件加速器的基础。
- **标准扩展（Standard Extensions）**：根据需要增加乘法、浮点、原子操作、向量等能力，以支持操作系统和各种通用软件的开发与运行。

以后读 RISC-V Manual，要养成一个习惯：

> **不要只翻译，要总结作者真正想表达的设计思想。**

这一段真正想表达的是：

> **RISC-V 采用"最小核心 + 按需扩展"的模块化设计。**

也就是说：

```
               RISC-V ISA
                    │
      ┌─────────────┴─────────────┐
      │                           │
 Base Integer ISA         Standard Extensions
 (最小可用核心)             (按需增加功能)
```
