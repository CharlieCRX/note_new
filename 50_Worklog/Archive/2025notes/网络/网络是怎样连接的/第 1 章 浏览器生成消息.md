## 1. 生成 HTTP 请求消息
### URL 和 URI
**URL 示例**

```bash
https://example.com:8080/path/to/file.html?query=test#section1
```

👉 这是一个 URL，因为它告诉你：

- 资源在哪（example.com:8080/path/to/file.html）
- 用什么协议（https）去访问

在实践中，当你在浏览器地址栏输入它时，浏览器就知道怎么去请求这个资源。

**URI**

URI 属于 URL 更高层次的抽象，一种字符串文本标准。

就是说，URI 属于父类，而 URL 属于 URI 的子类。URL 是 URI 的一个子集。

举例说明：

- **URI** = “这本书的唯一标识”
- **URL** = “图书馆书架 A 区第 5 行第 3 本”
- **URN** = “这本书的 ISBN 编号”

URI 是个大范围概念，URL 和 URN 是它的两种不同实现方式。URL 是“告诉你位置的 URI”。

对 URL 进行解析后，浏览器就获取到了Web 服务器和文件名，接下来就是根据这些信息来生成 HTTP 请求消息了。

## 2. HTTP 协议

- 面向对象：客户端和服务器
- API 标准：消息内容 + 步骤

### 发送请求消息

- 客户端请求服务器资源
- 请求格式：URI（Web资源的位置） + 方法（让Web服务器进行怎样的操作）

### 操作响应消息

- Web服务器响应客户端
- 响应格式：状态码 + 头字段 + 网页数据

响应消息会被发送回客户端，客户端收到之后，浏览器会从消息中读出所需的数据并显示在屏幕上。到这里，HTTP 的整个工作就完成了。
