# Worklog 使用说明

本目录用于记录**工作过程、学习轨迹与能力成长**，  
目标不是“留下详细记录”，而是 **形成可感知的进步感**。

---

## 目录结构说明

### Daily/
- 用于记录**当天发生了什么**
- 允许零碎、情绪化、不完整
- 不要求总结
- 原则：**写完即走，不回看**

### Weekly/
- 用于每周一次的轻量总结
- 回答三个问题：
  1. 这周推进了什么？
  2. 哪些地方卡住？
  3. 下周做什么？
- 这是维持“我在前进”的关键层

### Reviews/
- 用于阶段性 / 能力回顾
- 只记录一件事：**我现在比之前强在哪里**
- 是长期最有价值的部分

---

## 使用原则（非常重要）

- Daily 是“输入层”，允许混乱
- Weekly 是“整理层”，必须存在
- Reviews 是“沉淀层”，只写变化

**不要试图在一个文件里完成所有事情。**
