# 📘 CSAC 1PPS Disciplining（训练）机制说明文档

------

# 1. 系统概述

CSAC 内部实现了一个高精度的 1PPS 相位闭环控制系统，用于：

- 提升 1PPS 同步精度（<100 ns）
- 实现长期频率校准（5×10⁻¹³ 级别）

核心硬件模块：

> ✔ 高分辨率相位计（High-Resolution Phase Meter）

------

# 2. 相位测量机制（Phase Meter）

## 2.1 测量对象

相位计测量两个信号的时间差：

| 信号      | 来源               |
| --------- | ------------------ |
| 1PPS_CSAC | pin 10（内部输出） |
| 1PPS_EXT  | pin 9（外部参考）  |

------

## 2.2 测量方式

- 采样频率：**1 Hz（每秒一次）**
- 分辨率：**450 ps**

------

## 2.3 数学定义

定义相位误差：

```text
Phase = 1PPS_EXT - 1PPS_CSAC
```

------

## 2.4 符号定义

| 情况              | Phase 值  |
| ----------------- | --------- |
| CSAC 输出晚于参考 | Phase < 0 |
| CSAC 输出提前     | Phase > 0 |

------

# 3. 1PPS 训练（Disciplining）控制机制

## 3.1 控制目标

CSAC 通过闭环控制实现：

- 相位收敛（phase alignment）
- 频率收敛（frequency calibration）

------

## 3.2 控制结构

CSAC 使用双环控制：

```text
          Phase Meter
               ↓
        Phase Error (1 Hz)
               ↓
        Disciplining Algorithm
               ↓
        Steer（数字控制量）
               ↓
   Microwave Synthesizer (4.596 GHz)
               ↓
          TCXO + Physics Package
```

------

## 3.3 控制变量（Steer）

Steer 是核心控制量：

- 控制微波合成器频率
- 直接影响 CSAC 输出频率
- 用于同时修正：
  - 相位误差
  - 频率误差

------

## 3.4 控制特点

> ✔ CSAC 不是只调相位，而是“相位 + 频率联合控制”

------

# 4. 控制性能指标

系统最终可达到：

| 指标          | 性能    |
| ------------- | ------- |
| 1PPS 相位误差 | < 5 ns  |
| 频率稳定度    | 5×10⁻¹³ |

------

# 5. 控制算法（核心逻辑）

## 5.1 基本思想

每秒执行一次：

```text
1. 读取 Phase
2. 计算误差
3. 更新 Steer
4. 调整频率
```

------

## 5.2 控制形式（工程理解）

可以理解为：

```text
PI / PLL 控制系统：

Steer(k) = Steer(k-1) + Kp * Phase + Ki * ∑Phase
```

------

# 6. Disciplining 使能机制

通过 Mode Register 控制：

- Bit 4 (0x0010)

------

## 状态控制：

| 状态 | 含义              |
| ---- | ----------------- |
| 0    | 未启用            |
| 1    | 启用 disciplining |

------

## 特点：

- ✔ 非易失（断电保存）
- ✔ 上电自动恢复

------

# 7. 初始化流程（非常关键）

在开始训练前，系统执行：

## 7.1 初始化步骤

```text
1. 重置控制变量（Steer, filter）
2. 执行一次 1PPS sync
3. 将输出对齐到 <100 ns
4. 进入闭环控制
```

------

## 7.2 触发条件

- 第一次启用 disciplining
- 或 lock 状态恢复后重新进入

------

# 8. 异常处理机制

------

## 8.1 1PPS丢失（holdover）

当外部 1PPS 丢失：

```text
CSAC → holdover mode
```

行为：

- ✔ 保持最后 Steer 值
- ✔ 输出自由运行 TCXO
- ❌ 不进行 disciplining 更新

------

## 8.2 恢复策略

当 1PPS 回来：

| 条件         | 行为                        |
| ------------ | --------------------------- |
| drift < 1 μs | 继续原控制                  |
| drift > 1 μs | 重新 sync（但不重置控制器） |

------

## 8.3 强制重初始化

方法：

> disable disciplining → enable disciplining

------

# 9. Telemetry 输出行为

------

## 9.1 Phase 输出

- 输出最近一次测量值
- 精度：1 ns（显示精度）
- 来源：450 ps 分辨率量化

------

## 9.2 Phase 计算定义

```text
Phase = 1PPS_EXT - 1PPS_CSAC
```

------

## 9.3 方向解释

| Phase | 含义      |
| ----- | --------- |
| 正    | CSAC 提前 |
| 负    | CSAC 滞后 |

------

# 10. DiscOK 状态机

------

## 10.1 定义

DiscOK 表示“是否稳定锁定”

------

## 10.2 状态定义

| 值   | 含义     |
| ---- | -------- |
| 0    | 未锁定   |
| 1    | 锁定完成 |
| 2    | holdover |

------

## 10.3 判定规则

```text
DiscOK = 1 当且仅当：

|Phase| < threshold
持续时间 ≥ 2 × time constant
```

------

## 10.4 threshold 来源

- 用户设定（默认 ~20 ns）
- 可调（10 ~ 10000 ns）

------

# 11. 与其他模式互斥关系

CSAC 1PPS 模式互斥：

| 模式               | 是否可同时开启 |
| ------------------ | -------------- |
| Disciplining       | ❌              |
| Auto Sync          | ❌              |
| Phase Measure Mode | ❌              |

------

# 12. Time Constant（训练时间常数）

------

## 12.1 作用

控制闭环速度：

| 参数 | 作用           |
| ---- | -------------- |
| 小 τ | 快收敛，易抖动 |
| 大 τ | 慢收敛，稳定   |

------

## 12.2 典型范围

```text
10 s ~ 10000 s
```

------

## 12.3 工程意义

> τ 决定 PLL 带宽

------

# 13. 总结（最核心理解）

------

## ✔ 一句话理解 CSAC disciplining：

> CSAC 通过 1 Hz 的相位误差测量，驱动 Steer 控制微波频率，从而实现 1PPS 的闭环锁定系统。

------

## ✔ 三层结构：

| 层级        | 功能     |
| ----------- | -------- |
| Phase meter | 测误差   |
| Algorithm   | 算 Steer |
| Synthesizer | 修频率   |

------

## ✔ 两个目标：

- 短期：<100 ns 同步
- 长期：5×10⁻¹³ 频稳

## 示波器使用方法

现在 AD5932 能够从MSBOUT(8PIN) 输出 20KHz，每个波形持续周期为 5 个周期的方波。

那么应该怎么调整示波器抓波呢？

### 一、先算时间尺度

20 kHz：

```
T = 1 / 20000
  = 50 μs
```

5 个周期：

```
5 × 50 μs = 250 μs
```

所以示波器至少要看到：

```
250 μs
```

建议显示：

```
500 μs ～ 1 ms
```

方便观察前后空闲区。

### 二、示波器时基设置

先设置：

```
Time/Div = 50 μs/div
```

10 格屏幕：

```
50 μs × 10
= 500 μs
```

这样正好能看到：

```
空闲 + 5周期方波 + 空闲
```

如果波形太挤：

```
20 μs/div
```

如果想看全貌：

```
100 μs/div
```

### 三、垂直档位

例如：

#### 3.3V 方波

```
1V/div
```

#### 5V 方波

```
2V/div
```

然后调整 Position 让波形居中。

### 四、最关键：触发设置

如果只输出 5 个周期，必须正确触发。

#### 触发类型

选择：

```
Edge Trigger
```

------

#### Trigger Source

例如：

```
CH1
```

------

#### Trigger Edge

选择：

```
Rising
```

即：

```
↑
```

------

#### Trigger Level

假设：

```
0V ~ 3.3V
```

设置：

```
1.5V
```

附近即可。

## AD5932 如何通过起始频率和终止频率以及总输出时常设定参数

> 对，这样问题就完整了。
>
> 你实际上使用的是 AD5932 的：
>
> ```text
> Fixed Number of Output Waveform Cycles
> ```
>
> 模式。
>
> 此时真正决定每个频点停留时间的不是 Interval，而是：
>
> ```text
> CyclesPerStep
> =
> Interval × Multiplier
> ```
>
> 其中：
>
> | mclk_mult配置 | 实际倍数 |
> | ------------- | -------- |
> | 0             | 1        |
> | 1             | 5        |
> | 2             | 100      |
> | 3             | 500      |
>
> 即：
>
> ```text
> CyclesPerStep
> =
> Interval × K
> ```
>
> 其中：
>
> ```text
> K ∈ {1,5,100,500}
> ```
>
> ------
>
> # 正确时间模型
>
> 对于第 i 个频点：
>
> ```text
> Fi
> =
> Fstart + i·Δf
> ```
>
> 周期：
>
> ```text
> Ti
> =
> 1/Fi
> ```
>
> 输出：
>
> ```text
> CyclesPerStep
> =
> Interval × K
> ```
>
> 个周期。
>
> 因此该频点停留时间：
>
> ```text
> Tstep(i)
> =
> Interval × K × (1/Fi)
> ```
>
> ------
>
> 总扫频时间：
>
> # [ T_{scan}
>
> \sum_{i=0}^{N}
> \frac{Interval\times K}{F_i}
> ]$
>
> 提取公因子：
>
> # [ T_{scan}
>
> Interval \times K
> \times
> \sum_{i=0}^{N}
> \frac1{F_i}
> ]
>
> 定义：
>
> # [ S
>
> \sum_{i=0}^{N}
> \frac1{F_i}
> ]
>
> 则：
>
> # [ T_{scan}
>
> Interval \times K \times S
> ]
>
> 这就是 Solver 的核心公式。
>
> ------
>
> # Solver比之前容易很多
>
> 因为现在不再只有一个变量：
>
> ```text
> Interval
> ```
>
> 而是：
>
> ```text
> Interval × K
> ```
>
> 两个参数组合。
>
> 例如：
>
> 目标：
>
> ```text
> 50kHz → 100kHz
> 
> Duration=2ms
> 
> N=99
> ```
>
> 先计算：
>
> ```text
> S
> =
> Σ(1/Fi)
> ```
>
> 假设：
>
> ```text
> S = 1.386ms
> ```
>
> （实际值需要程序计算）
>
> ------
>
> 则需要：
>
> # [ Interval\times K
>
> # \frac{2ms}{1.386ms}
>
> 1.443
> ]
>
> ------
>
> 然后枚举：
>
> | Interval | K    | Product |
> | -------- | ---- | ------- |
> | 1        | 1    | 1       |
> | 1        | 5    | 5       |
> | 1        | 100  | 100     |
> | 1        | 500  | 500     |
> | 2        | 1    | 2       |
> | 2        | 5    | 10      |
> | 3        | 1    | 3       |
> | 4        | 1    | 4       |
>
> 等等。
>
> ------
>
> 你会发现：
>
> ```text
> 目标 Product = 1.443
> ```
>
> 最接近的是：
>
> ```text
> 1
> ```
>
> 或：
>
> ```text
> 2
> ```
>

>
> ```text
> Interval=1,K=1
> ```
>
> 或者：
>
> ```text
> Interval=2,K=1
> ```
>
> 计算误差后选最优。
>
> ------
>
> # Solver本质已经变了
>
> 以前（MCLK模式）：
>
> ```text
> 求 Interval
> ```
>
> 现在：
>
> ```text
> 求
> 
> Interval × K
> ```
>

>

>
> K=1

> ```

> 342
> ```

>
> K=5

> ```
>
> 实际：
>
> ```text
> 340
> ```
>
> ------
>
> K=100
>
> ```text
> Interval=3
> ```
>
> 实际：
>
> ```text
> 300
> ```
>
> ------
>
> K=500
>
> ```text
> Interval=1
> ```
>
> 实际：
>
> ```text
> 500
> ```
>
> ------
>
> 然后分别计算：
>
> # [ T_{actual}
>
> Interval\times K\times S
> ]
>
> 选择误差最小者。
>
> ------
>
> # 工程上更好的选择
>
> 我建议不要只比较误差。
>
> 比较规则：
>
> ```text
> 第一优先：
> 误差最小
> 
> 第二优先：
> Interval最大
> 
> 第三优先：
> K最小
> ```
>
> 原因：
>
> 例如：
>
> ```text
> Product=340
> ```
>
> 有两组解：
>
> ```text
> Interval=340
> K=1
> ```
>
> 和
>
> ```text
> Interval=68
> K=5
> ```
>
> 理论上时间一样。
>
> 但：
>
> ```text
> Interval=340
> ```
>
> 分辨率更高。
>
> 以后用户：
>
> ```text
> Duration变化0.1%
> ```
>
> 仍然能调整。
>
> 而：
>
> ```text
> Interval=68
> ```
>
> 量化误差会更大。
>
> 因此优先：
>
> ```text
> K=1
> ```
>

## 烤机功放需求分析

现在 AD5932 作为 DDS 可配置方波输出源，需要跟 12 个通道进行交互，实现界面可控的 12 通道 DDS 输出信号。

### 设备

- 1 个AD5932 DDS 输出。支持扫频和固定频率输出
- 12 个通道输出

### 需求分析

用户需要输入的是：

1.  DDS 输出形式

   - 扫频
   - 固定频率

2. 输出的通道数量

3. 同一通道下次输出时间

4. 每次输出信号的持续时间

例如，用户想要实现 10 个通道的输出，每次输出 2 ms 固定频率信号，同一通道的下次输出延迟为 500 ms。

这样拆分出来的参数为：

- 每个通道间隔 500/10 = 50ms 后，才能切换下一个通道进行输出
- 每次输出 2 ms 的 DDS 固定频率信号
- 依次打开 10 个通道

这样就在 50 ms 的间隔内，每个通道输出了 2ms 的 DDS 固定频率波。

所以第一个通道如果时间为 0 s 的时候，输出了 2ms 的信号，那么下一次发送其信号就是 500 ms。

同理，第二个通道在 50 ms 的时候，输出了 2 ms 的信号，那么下一次发送其信号时间就是 550 ms。

### 其他操作

-  中断信号输出：中途收到中断信号

  - 直到完成一轮后打断输出生成
  - 等待此通道输出完毕后立刻打断
  - 直接打断

- 由于每次切换通道时候，可能有 10 ms 的误差，所以需要加入时间判断：

  - 10个通道
  - 500ms间隔
  - 每次发送2ms

  这样每个通道间隔 50 ms 发送 2 ms 的DDS 信号。虽然有 10ms 的延迟，10 + 2 = 12 < 50 ms 的间隔 

已经将需求整理到了《DDS多通道功放控制系统接口设计 V2.3》中。

对于每个 N：

{N}

=

-

\right|

------

选择误差最小的一组。



## 正确时间模型

对于第 i 个频点：

```
Fi
=
Fstart + i·Δf
```

周期：

```
Ti
=
1/Fi
```

输出：

```
CyclesPerStep
=
Interval × K
```

个周期。

因此该频点停留时间：

```
Tstep(i)
=
Interval × K × (1/Fi)
```

------

总扫频时间：
$$
T_{scan}
=
\sum_{i=0}^{N}
\frac{Interval\times K}{F_i}
$$
提取公因子：
$$
T_{scan}
=
Interval \times K
\times
\sum_{i=0}^{N}
\frac1{F_i}
$$
定义：
$$
S
=
\sum_{i=0}^{N}
\frac1{F_i}
$$
则：
$$
T_{scan}
=
Interval \times K \times S
$$
这就是 Solver 的核心公式。

## 正确的计算流程

第一层：

寻找：
$$
N
$$
和
$$
\Delta f
$$
使终点频率误差没有。

------

第二层：

根据得到的：
$$
N
$$
计算：
$$
S
=
\sum_{i=0}^{N}
\frac1{F_i}
$$

------

第三层：

求：
$$
Interval
$$
和
$$
K
$$
使：
$$
T_{scan}
\approx
T_{target}
$$
定义：
$$
Cost
=
W_f\times Error_f
+
W_t\times Error_t
+
W_n\times |N-99|
$$
其中：

- $Error_f$：终点频率误差
- $Error_t$：扫频时间误差
- $N$：实际递增次数
- $99$：期望递增次数

这样 Solver 会自动得到：

```

终点频率就是Fstop

其次：
扫频时间最准

最后：
频点数尽量接近100
```

这才符合这个产品的真实需求：
