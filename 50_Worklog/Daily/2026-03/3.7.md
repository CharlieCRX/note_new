## 区分Domain和UseCase

就按照 servoV6 的架构来说：

- Domain 是业务模型。

  ```
  class Axis {
  public:
      void enable();
      void jog(Direction dir);
      void moveAbsolute(Position pos);
      void stop();
  
      Position position();
  };
  ```

  Domain 不关心：

  ```
  Modbus
  寄存器
  串口
  EtherCAT
  ```

- UseCase 是一个业务场景流程

设计原则上：

- UseCase 依赖 Domain 层的实现。

类比 PLC 控制：

- Domain：类似 PLC 定义的每个电机的控制和状态。例如 Y 轴电机的控制接口：

  ```Text
  使能轴Y电机
  轴Y手动速度
  轴Y定位速度
  轴Y回原点
  轴Y绝对位置清零
  轴Y绝对位置
  轴Y绝对定位触发
  ```

- UseCase：调用 PLC 接口实现某个功能。例如 Y 轴的点动流程：

  ```
  使能轴Y电机：ON
  设置 轴Y手动速度 = 5 mm/s
  轴Y点动正转 ON
  轴Y点动正转 OFF
  ```

  业务逻辑不应该依赖技术实现。

## ViewModel的存在价值

ViewModel 存在于 :

```
ui
 
viewModel

useCase
```

其主要作用是：

- 将 UI 状态管理从 UI 中抽离出来

UI只负责：

```
显示
用户输入
```

ViewModel负责：

```
状态
调用UseCase
处理结果
```

并且ViewModel接收 **设备状态**，然后整理为 **UI模型**。

另一个就是将电机状态转换为 UI 状态。
