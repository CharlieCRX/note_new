# 从`C_ADC_ADJUST`看逻辑

## 🧠 一、先建立“整体流程骨架”

你先不用看细节，先记住这 6 步：

```
① 参数准备
② 遍历增益
③ 遍历通道（设置硬件）
④ 采集数据（DMA）
⑤ 算幅值 → 算补偿
⑥ 保存补偿
```

👉 这是你后面理解所有细节的“主线”

------

## 🧩 二、第一段：函数入口在干嘛？

```C
int adc_adjust(int *adj_gainIDs, int *adj_chns, int signalHz, float ppMv)
```

👉 这几个参数的含义：

- `adj_gainIDs`：

  > ```C
  > #define GAIN_CAL_CNT (8)  // 增益档位
  > static const int g_supported_gain_db[GAIN_CAL_CNT] = { -20, -10, -6, 0, 6, 20, 30, 40 };
  > ```
  >
  > 当前系统支持的增益挡位为如上数组所示。
  >
  > 而这里的`adj_gainIDs`数组保存的就是启用了哪些增益。
  >
  > 例如下面的数组：
  >
  > ```C
  > int adj_gainIDs[] = {0, 0, 0, 1, 0, 0, 0, 1};
  > ```
  >
  > 就代表了：测试包括了增益 0、增益 40。
  >
  > 这里的`adj_gainIDs`的下标对应了 8 个增益挡位`g_supported_gain_db`下标。
  >
  > 而是否为1，代表了是否选择了此增益。

- `adj_chns`：此数组保存了每个通道是否要调整

- `signalHz`：输入信号的频率

- `ppMv`：输入信号的幅值

👉 提示你：
 这个函数其实是一个“校准任务调度器”
