## 开发驱动程序的注意

在开发Qt的时候，发现Qt弱C++。这样会导致自己的C++能力基本没有任何提升。

为了让自己学习到C++的真正精髓，所以准备通过新项目`servoV6`来学习C++。

所以倒推驱动程序的正确结构应该是：

```
Qt UI
   ↓
Application Layer
   ↓
Domain Layer（纯C++）
```

Qt 只负责：

```
按钮
输入
显示
```

业务逻辑：

```
全部写在纯 C++
```

例如：

Qt代码：

```C++
void MainWindow::onStartClicked()
{
    controller.startMotor();
}
```

真正逻辑：

```c++
class MotorController
{
public:
    void startMotor();
};
```

这样：

```
Qt只是壳
```

你的 **核心代码全是 C++**。

这样就不需要 **重写两遍**。

你不是想成为：

```
Qt工程师
```

你想成为：

```
工业软件架构工程师
```

核心能力应该是：

```
C++
架构设计
硬件控制系统
```

Qt 只是：

```
UI工具
```

## 项目架构的注意

### 1.最终架构推荐

当前：

```
UI
ViewModel
UseCase
Domain
```

稍微调整为：

```
Qt UI
   ↓
ViewModel (Qt适配层)
   ↓
Application (UseCase)
   ↓
Domain (纯业务模型)
   ↓
Infrastructure (设备/协议)
```

关键原则：

| 层             | 是否允许 Qt |
| -------------- | ----------- |
| UI             | ✅           |
| ViewModel      | ⚠️           |
| Application    | ❌           |
| Domain         | ❌           |
| Infrastructure | ❌           |

------

### 2.核心原则

Qt 的类型：

```C++
QObject
QString
QList
QTimer
signals
slots
```

**全部禁止进入 Application / Domain。**

所以 ViewModel 的职责变成：

```
Qt 世界  <->  C++ 世界 适配器
```

例如：

```C++
QString → std::string
QList → std::vector
signal → callback
```

### 3.推荐目录结构

我非常推荐你这样组织 servoV6：

```bash
servoV6

ui/
    qml/
    main.cpp

presentation/
    AxisViewModel.h

application/
    usecases/
        MoveAxisUseCase
        JogAxisUseCase

domain/
    entities/
        Axis
        GantryAxis

    value/
        Position
        Speed

    services/
        MotorService
        CommandProcessor

infrastructure/
    modbus/
        ModbusTransport
        PlcGateway

tests/
    domain_tests
    application_tests
```

重点：

```
domain
application
```

必须 **完全不依赖 Qt**。

### Step3

------

### Step4

```

```
