# PLC 软元件与地址映射模型

## 1. 本质

PLC 中的“变量”本质是：

> 对内部内存地址的抽象命名。

软元件地址（如 M1）是：

> PLC 内部物理内存编号。

变量名只是对地址的别名。

------

## 2. 映射关系

例如：

变量：

```
使能轴Y电机
```

软元件地址：

```
M1
```

逻辑关系：

```
使能轴Y电机 := TRUE
↓
M1 = 1
```

本质：

> 一个内存地址
> 两个名字

------

## 3. 类型与内存单位

- M 开头：位软元件
- 单位：1 bit

这意味着：

- BOOL 类型 → 位映射
- 并非“对象变量”
- 是直接对内存位的读写

------

## 4. 架构意义

理解地址映射模型后：

- Modbus 写寄存器 = 写 PLC 内存
- UI 操作 = 修改内部内存位
- 所有逻辑最终归结为“内存变化”

这使调试更清晰：

> 看地址，而不是看变量名。
