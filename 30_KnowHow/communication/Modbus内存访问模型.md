# Modbus 内存访问模型

## 1. 协议本质

Modbus 是一种：

> 主-从式的远程内存访问协议。

主站发起请求，
从站根据地址读取或写入内部内存区域。

------

## 2. 四种数据区的抽象意义

Modbus 并不是“不同类型数据”。

它只是对内部内存区域的划分。

| 类型              | 访问属性 | 本质               |
| ----------------- | -------- | ------------------ |
| Coils             | 读写     | 可写 bit 区域      |
| Discrete Inputs   | 只读     | 只读 bit 区域      |
| Holding Registers | 读写     | 可写 16-bit 数值区 |
| Input Registers   | 只读     | 只读 16-bit 数值区 |

本质是：

> 权限不同的内存分区。

------

## 3. 报文模型抽象

示例：

```
01 03 00 00 00 02 C4 0B
```

结构可以抽象为：

```
[从机地址] + [功能码] + [起始地址] + [数量] + [校验]
```

本质是：

> 远程内存读写命令。

------

## 4. 架构统一视角

Modbus + PLC 软元件模型结合后可以得出：

```
UI → Modbus → PLC 内存 → 变量映射 → 控制逻辑
```

所有控制问题最终可以归结为：

> 内存读写是否正确。
