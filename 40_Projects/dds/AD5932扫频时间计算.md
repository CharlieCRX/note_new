使用的是 AD5932 的：

```text
Fixed Number of Output Waveform Cycles
```

模式。

此时真正决定每个频点停留时间的不是 Interval，而是：

```text
CyclesPerStep
=
Interval × Multiplier
```

其中：

| mclk_mult配置 | 实际倍数 |
| ------------- | -------- |
| 0             | 1        |
| 1             | 5        |
| 2             | 100      |
| 3             | 500      |

即：

```text
CyclesPerStep
=
Interval × K
```

其中：

```text
K ∈ {1,5,100,500}
```

------

# 正确时间模型

对于第 i 个频点：

```text
Fi
=
Fstart + i·Δf
```

周期：

```text
Ti
=
1/Fi
```

输出：

```text
CyclesPerStep
=
Interval × K
```

个周期。

因此该频点停留时间：

```text
Tstep(i)
=
Interval × K × (1/Fi)
```

------

总扫频时间：

$T_{scan} =\sum_{i=0}^{N}
\frac{Interval\times K}{F_i}
]$

提取公因子：

$T_{scan} = Interval \times K
\times
\sum_{i=0}^{N}
\frac1{F_i}
$

定义：

$S = \sum_{i=0}^{N}\frac1{F_i}$

则：

$T_{scan}=Interval \times K \times S
$

这就是 Solver 的核心公式。

------

# Solver比之前容易很多

因为现在不再只有一个变量：

```text
Interval
```

而是：

```text
Interval × K
```

两个参数组合。

例如：

目标：

```text
50kHz → 100kHz

Duration=2ms

N=99
```

先计算：

```text
S
=
Σ(1/Fi)
```

假设：

```text
S = 1.386ms
```

（实际值需要程序计算）

------

则需要：

$Interval\times K\frac{2ms}{1.386ms}1.443$

------

然后枚举：

| Interval | K    | Product |
| -------- | ---- | ------- |
| 1        | 1    | 1       |
| 1        | 5    | 5       |
| 1        | 100  | 100     |
| 1        | 500  | 500     |

等等。

------

你会发现：

```text

```

最接近的是：

```text
1
```

```text
2
```

```text
Interval=1,K=1
```

或者：


计算误差后选最优。

# 一个非常好的求解策略

先计算：

$TargetProduct\frac{TargetDuration}{S}$

然后枚举 K：

```cpp
K={1,5,100,500}
```

计算：

$Intervalround(TargetProduct/K)$

例如：

```text
TargetProduct=342
```

------

K=1

```text
Interval=342
```

合法

实际：

```text
342
```

------

K=5

```text
Interval=68
```

实际：

```text
340
```

------

K=100

```text
Interval=3
```

实际：

```text
300
```

------

K=500

```text
Interval=1
```

实际：

```text
500
```

------

然后分别计算：

$T_{actual}Interval\times K\times S$

选择误差最小者。

# 工程上更好的选择

我建议不要只比较误差。

比较规则：

```text
第一优先：
误差最小

第二优先：
Interval最大

第三优先：
K最小
```

原因：

例如：

```text
Product=340
```

有两组解：

```text
Interval=340
K=1
```

和

```text
Interval=68
K=5
```

理论上时间一样。

但：

```text
Interval=340
```

分辨率更高。

以后用户：

```text
Duration变化0.1%
```

仍然能调整。

而：

```text
Interval=68
```

量化误差会更大。

因此优先：

```text
K=1
```
