# 🧠 今日核心知识（必须掌握）

------

## 1️⃣ `std::unordered_map` 本质

> 👉 **一个 Key → Value 的哈希表（无序，查找快）**

✔ 适合：寄存器地址 → 寄存器值
❌ 不保证顺序

------

## 2️⃣ `operator[]` 是“危险操作”

```cpp
registers[addr]
```

> ❗本质：**查找 + 插入**

✔ key 不存在 → 自动创建（默认值 0）
🔥 会导致：**读操作改变状态（严重 bug 来源）**

------

## 3️⃣ `find()` 是“安全查询”

```cpp
auto it = registers.find(addr);
```

> 👉 只查，不改

✔ 不存在 → 返回 `end()`
✔ 不会污染数据

------

## 4️⃣ `iterator`（迭代器）

```cpp
it->first   // key
it->second  // value
```

> 👉 本质：**指向容器元素的位置（类似安全指针）**

------

## 5️⃣ `auto` 的意义

```cpp
auto it = ...
```

> 👉 **让编译器自动推导类型**

✔ 减少冗长类型（特别是 iterator）
✔ 现代 C++ 必用习惯

------

## 6️⃣ `::`（作用域解析符）

```cpp
RegisterManager::read
```

> 👉 表示：**这个函数属于哪个类/命名空间**

✔ 不写 → 变成普通函数 ❌
✔ 写了 → 成员函数 ✅

------

## 7️⃣ `.h vs .cpp` 的本质

```text
.h   → 接口（声明）
.cpp → 实现（定义）
```

> 👉 **解耦接口和实现（工程能力核心）**

------

## 8️⃣ 成员函数 `const`

```cpp
uint16_t read(...) const
```

> 👉 **承诺：不会修改对象状态**

✔ 可以 `find()`
❌ 不能 `registers[addr] = ...`

------

## 9️⃣ 架构认知（最重要）

你写的不是：

```text
一个 unordered_map
```

你写的是：

```text
PLC寄存器系统（Domain模型）
```

👉 `unordered_map` 只是实现细节

------

# ⚠️ 今日最大坑（必须记住）

> ❗ **不要在“读操作”里用 `operator[]`**

否则你会得到：

```text
系统被悄悄修改，但你不知道
```

👉 工业系统里这是非常危险的

------

# 🧭 今日最重要的一句话（请记住）

> 🔥 **“数据结构 ≠ 业务模型”**

- map 是工具
- RegisterManager 才是模型
