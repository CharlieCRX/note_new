# 今天目标（只做一件事）

```
让测试可以运行
```

也就是：

```
./tests
PASS
```

只要做到这一点，你整个项目以后都可以 **TDD开发**。

------

# 一、安装 GoogleTest

Linux 下执行：

```bash
sudo apt install libgtest-dev
```

安装后源码在：

```
/usr/src/gtest
```

Ubuntu 的 gtest 需要编译一次：

```bash
cd /usr/src/gtest
sudo cmake .
sudo make
sudo cp lib/*.a /usr/lib
```

完成后系统就有：

```
libgtest.a
libgtest_main.a
```

------

# 二、调整你的项目结构

建议稍微规范一点（以后不会乱）：

```
plc_simulator
│
├── src
│   └── main.cpp
│
├── tests
│   └── test_main.cpp
│
├── CMakeLists.txt
│
└── build
```

------

# 三、写一个最简单测试

创建：

```
tests/test_main.cpp
```

内容：

```cpp
#include <gtest/gtest.h>

TEST(SampleTest, BasicTest)
{
    EXPECT_EQ(1 + 1, 2);
}
```

------

# 四、修改 CMakeLists.txt

你的 CMakeLists 改成：

```cmake
cmake_minimum_required(VERSION 3.10)

project(plc_simulator)

set(CMAKE_CXX_STANDARD 17)

enable_testing()

add_executable(plc_simulator
    src/main.cpp
)

add_executable(tests
    tests/test_main.cpp
)

target_link_libraries(tests
    gtest
    gtest_main
    pthread
)

add_test(NAME run_tests COMMAND tests)
```

------

# 五、重新编译

进入 build：

```bash
cd build
cmake ..
make
```

------

# 六、运行测试

```bash
./tests
```

你应该看到：

```
[==========] Running 1 test
[ RUN      ] SampleTest.BasicTest
[       OK ] SampleTest.BasicTest
[==========] 1 test from 1 test suite ran
[  PASSED  ] 1 test
```

成功 🎉
