------

# 一、PLC Simulator 总体架构

核心思想：

```
协议层 ≠ 业务层 ≠ 设备模拟
```

系统结构：

```
                 Client
                   │
                   │ Modbus TCP
                   ▼
             ┌──────────────┐
             │   TcpServer   │
             └──────┬───────┘
                    │
                    ▼
             ┌──────────────┐
             │ ModbusParser  │
             └──────┬───────┘
                    │
                    ▼
            ┌─────────────────┐
            │ RegisterManager │
            └──────┬──────────┘
                   │
        ┌──────────┴──────────┐
        ▼                     ▼
 SimulationEngine        UI Monitor
        │
        ▼
     Devices
 (Motor / Temp)
```

一句话理解：

```
网络 → 协议 → 寄存器系统 → 设备模拟
```

寄存器是整个系统的 **中心数据模型**。

# 三、核心模块设计

我们一个一个看。

------

# 1 RegisterManager（核心数据模型）

这是 **整个系统的中心**。

职责：

```
管理 PLC 寄存器
提供读写接口
```

结构：

```cpp
class RegisterManager
{
public:
    uint16_t read(uint16_t addr) const;

    void write(uint16_t addr, uint16_t value);

    std::vector<uint16_t> readBlock(
        uint16_t start,
        uint16_t count);

private:

    std::unordered_map<uint16_t, uint16_t> registers;
};
```

说明：

```
地址 → 值
```

例如：

```
40001 → 100
40002 → 50
```

------

# 四、MemoryMap（寄存器定义）

RegisterManager 只负责 **存储**。

MemoryMap 负责 **定义寄存器含义**。

例如：

```
40001 motor_enable
40002 motor_speed
40003 motor_position

30001 temperature
```

设计：

```cpp
struct RegisterInfo
{
    uint16_t addr;
    std::string name;
    std::string description;
};
```

MemoryMap：

```cpp
class MemoryMap
{
public:

    void addRegister(RegisterInfo reg);

    std::optional<RegisterInfo>
    find(uint16_t addr) const;

private:

    std::unordered_map<uint16_t, RegisterInfo> map;
};
```

# 五、TcpServer（网络层）

职责：

```
接收 TCP 连接
接收数据
发送数据
```

接口：

```cpp
class TcpServer
{
public:

    void start(int port);

private:

    void acceptLoop();

    void handleClient(int client_fd);
};
```

流程：

```
accept
  ↓
recv
  ↓
parser
  ↓
send
```

------

# 六、ModbusParser（协议层）

职责：

```
解析 Modbus TCP
生成请求对象
```

设计：

```
网络数据 → Request
```

结构：

```cpp
struct ModbusRequest
{
    uint8_t function;
    uint16_t addr;
    uint16_t count;
};
```

解析器：

```cpp
class ModbusParser
{
public:

    ModbusRequest parse(
        const std::vector<uint8_t>& data);
};
```

------

# 七、ResponseBuilder

构建响应。

```cpp
class ResponseBuilder
{
public:

    std::vector<uint8_t>
    buildReadResponse(
        const std::vector<uint16_t>& values);
};
```

------

# 八、SimulationEngine（设备模拟核心）

职责：

```
周期更新设备状态
```

例如：

```
电机转动
温度变化
```

设计：

```cpp
class SimulationEngine
{
public:

    void start();

    void addDevice(
        std::shared_ptr<Device> device);

private:

    void loop();
};
```

------

# 九、Device 抽象类

设备模拟统一接口。

```cpp
class Device
{
public:

    virtual ~Device() = default;

    virtual void update(double dt) = 0;
};
```

说明：

```
dt = 时间间隔
```

------

# 十、MotorSimulator

模拟电机。

寄存器：

```
40001 enable
40002 speed
40003 position
```

实现：

```cpp
class MotorSimulator : public Device
{
public:

    MotorSimulator(RegisterManager& regs);

    void update(double dt) override;

private:

    RegisterManager& registers;
};
```

逻辑：

```
if enable
    position += speed * dt
```

# 十三、系统运行流程

启动流程：

```
main
 │
 ├ RegisterManager
 │
 ├ SimulationEngine
 │
 ├ TcpServer
 │
 └ UI
```

通信流程：

```
Client
   │
TCP
   │
TcpServer
   │
ModbusParser
   │
RegisterManager
   │
ResponseBuilder
   │
Client
```

设备模拟：

```
SimulationEngine
   │
每 50ms
   │
update(device)
```

------

# 十四、最小可运行版本（你第一阶段只做这个）

你第一阶段只需要做：

```
RegisterManager
TcpServer
ModbusParser
```

先不做：

```
UI
Simulation
```

最小系统：

```
Client → TCP → Modbus → RegisterManager
```

这样 **2周就能跑起来**。

# 十五、这个架构的好处

这个结构实际上是 **工业软件标准结构**：

优点：

### 1 网络和协议解耦

```
TcpServer 不知道 Modbus
```

------

### 2 协议和设备解耦

```
Modbus 不知道电机
```

------

### 3 寄存器是唯一数据源

```
设备
通信
UI
```

全部通过：

```
RegisterManager
```

------

### 4 易扩展

未来你可以加：

```
OPC UA
Web UI
脚本模拟
```

都不用改核心。
