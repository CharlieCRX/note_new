------

# 📘 双轴同步控制验收清单

------

# 0 同步测试准备（新增）

在任何同步测试前必须执行：

### 0.1 位置预对齐

操作：

```
Move X1
Move X2
```

使两轴满足：

```
ABS(X1_Pos - X2_Pos) ≤ 0.1 mm
```

目的：

避免初始位置误差影响测试结果。

------

### 0.2 确认系统静止

确认：

```
X1_Vel = 0
X2_Vel = 0
```

并且：

```
X1_State = IDLE
X2_State = IDLE
```

------

# 1 同步启动测试

原方案的问题：

> “同PLC周期启动”在现场其实很难直接观察。

更可靠的方法是：

**启动时间差验证**

------

## 操作

执行：

```
Sync_MoveAbs = 100 mm
Speed = 50 mm/s
```

------

## 监控变量

```
X1_State
X2_State

X1_Pos
X2_Pos
```

------

## 记录

记录：

```
X1进入RUNNING的时间
X2进入RUNNING的时间
```

------

## 判定

允许差值：

```
≤ 1 PLC scan cycle
```

或：

```
≤ 5 ms
```

------

# 2 同步运动行为测试

目标：

确认 **同步过程中不会逐渐漂移**

------

## 操作

执行：

```
MoveAbs = 1000 mm
Speed = 100 mm/s
```

------

## 监控变量

```
X1_Pos
X2_Pos
```

计算：

```
Sync_Error = ABS(X1_Pos - X2_Pos)
```

------

## 记录

在运行过程中记录：

```
Sync_Error
Sync_Error_Max
```

------

## 判定

| 项目     | 标准     |
| -------- | -------- |
| 最大误差 | ≤ 1 mm   |
| 平均误差 | ≤ 0.5 mm |

------

# 3 同步漂移测试（新增关键测试）

这是很多系统会失败的地方。

------

## 操作

连续执行：

```
MoveAbs +500 mm
MoveAbs -500 mm
```

循环：

```
5 次
```

------

## 监控

记录：

```
X1_Pos
X2_Pos
```

最终误差：

```
Final_Error = ABS(X1_Pos - X2_Pos)
```

------

## 判定

```
Final_Error ≤ 0.5 mm
```

如果逐渐变大：

说明：

```
同步补偿不存在
```

------

# 4 同步误差监测

建议将逻辑从：

```
X1_Pos - X2_Pos
```

改为：

```
Sync_Error = ABS(X1_Pos - X2_Pos)
```

并持续监控：

```
Sync_Error_Max
```

------

# 5 超差保护测试（增强）

原方案：

```
拖动其中一轴
```

实际更安全方法：

### 方法1：降低一轴速度

```
X1_Speed = 100 mm/s
X2_Speed = 50 mm/s
```

观察：

```
Sync_Error
```

------

### 方法2：人为阻挡（当前无法测量）

轻微阻挡其中一轴。

------

## 验证

系统应触发：

```
Sync_Alarm
```

并执行：

```
Stop X1
Stop X2
```

------

# 6 同步结束验证（优化）

原方案只看状态。

建议增加：

**位置一致性验证**

------

## 操作

执行：

```
Sync_MoveAbs = 1000 mm
```

------

## 结束时验证

```
X1_State = FINISHED
X2_State = FINISHED
```

并且：

```
ABS(X1_Pos - X2_Pos) ≤ 0.5 mm
```

------

# 7 软限位逻辑（保持）

验证：

```
触发 SoftLimit
```

系统行为：

```
Stop Axis
```

------

## 撤离测试

执行：

```
Jog Reverse
```

应允许：

```
退出软限位区域
```

------

# 8 必须记录的变量（优化）

建议现场记录：

```
X1_Pos
X2_Pos

X1_Vel
X2_Vel

Sync_Error
Sync_Error_Max

X1_State
X2_State
```

------

# 9 验收最关键判断标准

真正的同步系统必须满足：

### 1 启动同步

```
启动时间差 ≤ 1 scan cycle
```

------

### 2 运行稳定

```
Sync_Error ≤ 1 mm
```

------

### 3 无累计漂移

```
多次往返误差 ≤ 1 mm
```

------

### 4 超差保护

```
Sync_Error > Limit → Stop
```

------

# 10 最重要的一点（工程经验）

很多系统看起来同步，但其实只是：

```
同时启动
```

真正同步必须具备：

```
实时误差控制
```

否则在长行程运动时：

```
误差会持续累积
```

------

# 最后给你一个建议（非常实用）

在验收时，**最重要的不是看代码，而是画一条曲线**：

```
Sync_Error vs Time
```

如果曲线：

```
稳定在一个小范围
```

说明同步成立。

如果曲线：

```
不断漂移
```

说明系统只是 **假同步**。
