------

#  软限位逻辑修复复测验收报告

------

## 一、问题回顾

此前存在的问题为：

- 触碰软限位后
- 绝对定位状态未复位
- 系统进入“假运行状态”
- 无法执行新的定位指令
- 无法脱离软限位区域

问题本质属于：

> 运动异常终止未完成状态闭环处理

------

## 二、本次整改内容确认

经过与开发人员沟通，本次整改主要涉及以下三项优化：

### 1️⃣ 运动状态闭环修复

当出现以下任一情况时：

- 到位信号
- 正软限位触发
- 负软限位触发
- 伺服报警

统一执行：

```text
绝对定位中 = OFF
相对定位中 = OFF
```

验证结果：

- 触发软限位后，“绝对定位中”状态自动复位
- 系统恢复为可操作状态
- 不再出现假运行状态

✅ 状态生命周期闭环已完成

------

### 2️⃣ 点动逻辑放开

整改前：

- 触发软限位后
- 无法进行点动

整改后：

- 允许反向点动脱离软限位
- 禁止继续朝触发方向运动

验证结果：

| 测试项                   | 结果     |
| ------------------------ | -------- |
| 触发负软限位             | 正常停止 |
| 正向点动（继续向负方向） | 被禁止   |
| 反向点动（回正方向）     | 允许执行 |
| 脱离限位后状态恢复       | 正常     |

✅ 方向限制逻辑正确
✅ 支持安全撤离机制

------

### 3️⃣ 绝对定位逻辑调整说明

针对：

> 为什么触发软限位后，不能通过绝对定位回到合法区域？

开发给出的设计逻辑为：

- 触发软限位视为异常场景
- 在执行绝对定位前，会进行目标位置合法性校验
- 若目标位置超出软限位范围，则拒绝执行

即：

```text
IF 目标位置 < 负软限位 OR 目标位置 > 正软限位 THEN
    禁止绝对定位
END_IF
```

因此：

- 只能通过点动模式脱离软限位
- 不允许通过绝对定位“跨越非法区间”

这是出于：

> 安全优先设计策略（避免误动作）

验证结论：

- 合法区间内的绝对定位 → 可执行
- 非法区间目标 → 被正确拒绝
- 已触发限位状态下 → 需先点动撤离

✅ 逻辑符合安全控制原则

------
