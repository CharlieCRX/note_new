# 🔴 第一阶段：单轴基础运动验收（最先做）

目标：

> 保证“单轴完全正确”，再谈双轴

------

## 1️⃣ 点动测试

测试项：

- 正向点动
- 反向点动
- 松手立即停止
- 限位触发停止

验证点：

- 速度单位是否为 mm/s 或 °/s
- 是否存在加减速异常
- 点动状态是否进入 RUNNING

你记录：

- 指令速度
- 实际速度
- 停止延迟

工时：0.5 天

------

## 2️⃣ 相对位置移动

测试：

- 相对 +10mm
- 相对 -10mm
- 小位移 0.1mm
- 大位移

检查：

- 运动完成是否进入 FINISHED
- 是否真的到位（误差 < 阈值）
- 不能以“命令下发完成”为结束

工时：0.5 天

------

## 3️⃣ 绝对位置移动（基于相对坐标系）

测试：

- 设置相对原点
- 移动到 0
- 再移动到 50

确认：

```text
Relative = Absolute - Offset
```

是否计算正确。

工时：0.5 天

------

# 🟠 第二阶段：机械参数换算验证

这一步非常关键。

你必须验证：

> PLC 的物理单位换算是否正确

测试方法：

1. 设置丝杆导程 = 10mm/rev
2. 设置减速比 = 5
3. 下发 10mm

理论电机转数应该：

```
10mm / 10mm = 1rev
电机 = 1rev × 5 = 5rev
```

你要：

- 读驱动器转数
- 看是否符合理论

这个是很多系统出错的地方。

工时：1 天

------

# 🔴 第三阶段：双轴同步控制（核心）

你重点关注 3 件事：

------

## 1️⃣ 同步启动

测试：

- 同步模式启动
- 两轴方向相反
- 速度相同

问供应商：

> 是同周期置位吗？

你观察：

- 是否存在肉眼可见延迟

------

## 2️⃣ 运行中误差监测周期

关键问题：

- 误差比较在什么周期执行？
- 扫描周期多少？

你可以要求：

> 打开在线监视，显示误差变量

看误差是否实时更新。

------

## 3️⃣ 强制制造误差

方法：

- 手动拖动其中一轴（低速）
- 或人为施加负载

确认：

- 是否触发 ALARM
- 是否执行受控停机

工时：2 天

------

# 🔴 第四阶段：超差补偿流程

这是你验收难点。

流程应该是：

```
超差 → 进入 ALARM → 禁止同步 → 允许单轴补偿
```

你测试：

1. 触发超差
2. 是否禁止同步指令
3. 是否只能单轴运行
4. 补偿完成后是否恢复同步模式

工时：2 天

------

# 🟡 第五阶段：限位与组合限位

测试：

- 单轴最大/最小限位
- X-Z 动态限位

你制造边界场景：

- X 在极限位置
- 再移动 Z
- 看是否被拒绝

工时：1.5 天

------

# 🟡 第六阶段：状态机完整性

你只关注状态转换是否合法：

```
IDLE → RUNNING → FINISHED
IDLE → RUNNING → ALARM
ALARM → ALARM_CLEARED → IDLE
```

你要确认：

- 是否可以在 ALARM 状态下下发运动（必须禁止）
- 是否存在跳状态现象

工时：1.5 天

------

# 🟢 第七阶段：坐标与原点体系

测试：

1. 设置当前位置为相对原点
2. 清除相对原点
3. 执行绝对清零
4. 验证三个位置量是否符合约定

特别关注：

> 绝对清零后是否提示重新设置相对原点

工时：1.5 天
