# 1. 验收清单（双轴同步 — PLC 逻辑版）

每项都给出目标、检查点以及判定标准（通过/不通过）。

## 1. 前置准备（必须全部通过）

- PLC 与驱动器通信正常（MODBUS/Profinet 等）：读取驱动器状态寄存成功。
- 所有安全回路正常：急停未触发、门锁闭合、使能允许。
- 机械/电气检查：丝杆/减速/编码器连接、限位开关就位。
- 参数加载：每轴机械参数（传动方式、丝杆导程、减速比、轴类型）已写入 PLC 并可读取。

判定：能读取所有关键状态寄存器且值合理 → 通过。

## 2. 单轴功能（X1、X2 及其它单轴逐个）

对每轴执行下列检查（逐项记录）：

- 点动（正/反）：
  - 操作：设置点动速度（物理单位），触发点动正/反位（写 bit）。
  - 检查：驱动器实际速度 `fActVelocity` 与设定方向一致；位置以物理单位持续变化；无报警。
  - 判定：在点动命令下瞬时速度与方向正确，释放后停止并进入 IDLE。
- 定位（相对 / 绝对）：
  - 操作：相对移动给定增量（mm 或 °）；绝对移动到给定相对坐标。
  - 检查：运动进入 RUNNING，驱动器到位（InPosition/TargetReached）或速度=0且误差小于阈值才判 FINISHED；绝对/相对位置显示正确。
  - 判定：按位置误差阈（例如 ≤0.1mm / ≤0.1°，由项目定义）到位。
- 限位动作：
  - 将轴移动至接近最小/最大物理极限，确认限位触发并生成报警或被安全停止。
  - 判定：限位触发时 PLC 进入 ALARM 并禁止新运动。

## 3. 双轴同步（X1 / X2）

- 同步启动条件检查：
  - 确认 `轴X联动使能` 为 ON，且两轴均处于可使能状态（bPowerState）。
- 同步运动命令（性能检查）：
  - 操作：以物理速度设定两个轴相同速度、方向相反（PLC 内自动换算下发）。
  - 检查：两轴 `fActVelocity` 同步（幅值相等，符号相反）且相对位置随时间按预期变化。
  - 判定：在正常负载下运行稳定，位置同步误差在可接受阈值内（项目定义，例如 ≤0.2mm）。

## 4. 同步超差检测与处理

- 人为制造超差：
  - 操作：在同步运行中短时改变其中一轴的速度或用外力干涉，造成物理位置偏差超过阈值。
  - 检查：PLC 应实时检测到 `abs(Pos_X1 - Pos_X2) > 超差阈值`，立即执行：受控停机、禁止同步模式、置 ALARM（或设定错误类型），并进入补偿/故障流程。
  - 判定：超差触发动作（停机/报警）在阈值被超出后的控制周期内完成（例如 ≤1 PLC 扫描周期或项目要求时间）；禁止新同步命令。

## 5. 单轴补偿流程（在超差后）

- 切换逻辑：
  - PLC 能将系统从“同步 mode”切换为“单轴补偿 mode”，目标由物理差值计算（例如将后退/前进少的轴补偿差值）。
- 补偿约束：
  - 补偿过程受限位、安全规则、报警优先级约束；若补偿无法在限位内完成，进入 ALARM。
- 判定：补偿结束后两轴相对差值恢复至 ≤补偿阈值并可重新允许同步（需人工确认或自动判断，按项目规则）。

## 6. 状态机与报警

- 验证 PLC 对每次运动正确上报状态（IDLE / RUNNING / FINISHED / ALARM / ALARM_CLEARED）。
- 验证报警优先级：任何 ALARM 时禁止新运动、触发受控停机。
- 验证 ALARM_CLEARED 流程：alarm 源清除后，上位或用户命令解除报警，PLC 切换为 ALARM_CLEARED。
