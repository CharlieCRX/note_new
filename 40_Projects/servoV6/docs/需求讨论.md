# 单轴需求汇总

为了方便后续确定真正的项目框架，现在跟你讨论下我当前的需求。首先我们就拿着单轴控制的要求来：

这里也包括了一个旋转轴，并且展示信息类似，就是以度数为单位，而不是mm

## 单轴控制需求（YZ）

展示信息：

- 当前位置以及设定的软限位范围
- 当前速度
- 电机状态
- 设置的定位速度
- 设置的点动速度
- 报警状态以及提示信息（红绿黄）
- 电机划分（Y和Z周电机）

控制：

- 点动和定位模式的切换

- 设置点动速度

- 设置定位速度

- 设置定位距离

- 回原点按钮

- 点动前进/位置前进

- 点动后退/位置后退

- 电机切换（Y和Z的切换）

- 急停按钮

其他的按钮信息：

- 登录用户提示
- 上位机电池状态
- 配置信息
- PLC 连接状态

现在这个控制软件要适配在安卓掌机上，需要做摇杆和移动的适配。

# X1X2联动轴需求

讲完了YZ单轴运动，下面我们聊一下X1X2双轴联动的逻辑。

## 双轴X1X2控制需求

状态展示信息：

- 主轴X1和从轴X2的当前位置以及设定的软限位范围
- X1、X2的当前速度
- X1、X2的电机状态
- X1、X2的设置的定位速度
- X1、X2的设置的点动速度
- X1、X2的报警状态以及提示信息（红绿黄。同时提示是否超差）
- X1X2是否开启了联动模式

控制信息：

- 点动和定位模式的切换
- 设置X1X2联动龙门结构点动速度

- 设置X1X2定位速度

- 设置X1X2定位距离

- X1X2回原点按钮

- X1X2点动前进/位置前进

- X1X2点动后退/位置后退

- 联动模式切换（单独控制X1或X2）

- 急停按钮（急停X1X2）

# 预期目标

- 首先就是分层

  ```
  UI (Qt/QML)
  
  Presentation (ViewModel)
  
  Application (UseCase)
  
  Domain (业务模型)
  
  Infrastructure (通信/驱动)
  ```

- 然后偏重C++ 和 UI 开发。

  ```
  能用 STL 就不用 for
  能用 RAII 就不用 new
  能用对象建模就不用 if
  ```

  | 层             | 是否允许 Qt |
  | -------------- | ----------- |
  | UI             | ✅           |
  | ViewModel      | ⚠️           |
  | Application    | ❌           |
  | Domain         | ❌           |
  | Infrastructure | ❌           |

- 能否实现设计一个 **“硬件抽象层 (HAL)”** 。

  **目标：** 实现逻辑与硬件分离，无论底层是 PLC 还是 ARM 驱动，上位机 Qt 代码一套都不用改。这体现了 JD 中要求的 **“业务抽象”** 能力 。
