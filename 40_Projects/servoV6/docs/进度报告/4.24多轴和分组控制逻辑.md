# 📘 一、问题定义（为什么需要多轴 + 分组）

------

## 🎯 当前系统能力

```text
✔ 单轴控制（Axis）
✔ 单系统（单PLC）
✔ 完整闭环（Command → PLC → Feedback）
```

------

## 🚧 新需求

```text
1. 多轴（Y / Z / R / X1 / X2）
2. 多分组（多套电机系统 / 多PLC）
```

------

## 🎯 目标

```text
实现：

👉 多轴复用同一控制逻辑
👉 多分组独立运行
👉 UI可自由切换控制对象
```

------

# 🧠 二、核心抽象（最重要）

------

# 🟢 1️⃣ Axis（轴）

```text
Axis = 单个执行单元（电机/伺服）
```

------

## 特征：

```text
✔ 有状态（Idle / Moving / Error）
✔ 有命令（intent）
✔ 有反馈（feedback）
✔ 可被控制（Move / Jog / Stop）
```

------

## ❗关键约束：

```text
Axis 不区分：

❌ Y / Z / R
❌ 不关心PLC地址
❌ 不关心分组
```

------

👉 本质：

```text
Axis = 抽象物理模型
```

------

# 🟡 2️⃣ AxisId（轴标识）

```text
AxisId = 轴的身份（Y / Z / R / X1 / X2）
```

------

## 作用：

```text
✔ 标识“操作哪个轴”
✔ 用于索引 Axis 实例
✔ 用于 Driver 映射寄存器
```

------

👉 本质：

```text
AxisId = 逻辑索引（不是业务逻辑）
```

------

# 🟠 3️⃣ AxisRepository（多轴容器）

```text
AxisRepository = 一组轴的集合
```

------

## 职责：

```text
✔ 管理多个 Axis 实例
✔ 提供 AxisId → Axis 映射
✔ 保证每个轴独立状态
```

------

## 语义：

```text
AxisRepository = “一个系统中的所有轴”
```

------

# 🔵 4️⃣ Group（分组）

```text
Group = 一套完整的控制系统
```

------

## 包含：

```text
✔ 多个 Axis（Y/Z/R/X1/X2）
✔ 一个 Driver（PLC连接）
✔ 一套 Orchestrator
✔ 一套运行状态
```

------

## ❗关键理解：

```text
Group ≠ 轴集合

Group = 一个“完整运行系统”
```

------

👉 本质：

```text
Group = System Instance（系统实例）
```

------

# 🟣 5️⃣ SystemContext（分组实体）

```text
SystemContext = Group 的运行载体
```

------

## 包含：

```text
✔ AxisRepository
✔ Driver（PLC）
✔ Orchestrator（可选）
✔ 状态（运行时）
```

------

## 语义：

```text
SystemContext = 一套“可运行的控制系统”
```

------

------

# 🧭 三、业务语义拆解（多轴 vs 分组）

------

# 🟢 多轴（Axis 维度）

------

## 🎯 定义

```text
多轴 = 在同一系统中控制多个 Axis
```

------

## 特点

```text
✔ 共用同一 Driver（同一PLC）
✔ 共用同一 SystemContext
✔ Axis之间相互独立（单轴场景）
```

------

## 示例

```text
Group1：

Y轴
Z轴
R轴
```

------

## 本质

```text
多轴解决：

👉 “操作哪个执行单元”
```

------

------

# 🟡 分组（Group 维度）

------

## 🎯 定义

```text
分组 = 多套独立系统并存
```

------

## 特点

```text
✔ 每组有独立 PLC
✔ 每组有独立 AxisRepository
✔ 每组独立运行（互不影响）
```

------

## 示例

```text
Group1 → PLC A → Y/Z/R
Group2 → PLC B → Y/Z/R
Group3 → PLC C → Y/Z/R
```

------

## 本质

```text
分组解决：

👉 “操作哪一套系统”
```

------

------

# 🔁 四、系统运行模型（核心）

------

## 单 Group 运行

```text
tick():

1. Orchestrator.step()
2. Axis → 生成 intent
3. Driver.send(axis)
4. PLC执行
5. 产生 feedback
6. Axis.applyFeedback
```

------

## 多 Group 运行

```text
for each group:
    tick(group)
```

------

👉 本质：

```text
多个系统并行运行
```

------

------

# 🖥 五、UI 交互语义

------

## 🎯 UI 控制维度

```text
UI选择：

1. 当前 Group
2. 当前 Axis
```

------

## 操作流程

```text
用户选择：

Group2 + Z轴

→ ViewModel(Z, Group2)

→ 发起控制
```

------

## ❗关键约束

```text
UI 不直接控制：

❌ PLC
❌ Driver
❌ Axis
```

------

👉 只能：

```text
通过 ViewModel → UseCase → SystemContext
```

------

# 🧠 六、分层职责（结合多轴 + 分组）

------

## 🟣 Domain

```text
定义 Axis 行为
```

------

## 🔵 Application

```text
执行操作（UseCase）
编排流程（Orchestrator）

👉 接收 AxisId
```

------

## 🟠 Infrastructure

```text
实现 Driver

👉 AxisId → 寄存器映射
👉 Group → PLC连接
```

------

## 🟡 Presentation

```text
选择：

👉 当前 Group
👉 当前 Axis
```

------

## 🟢 UI

```text
提供操作入口
```

------

------

# ⚠️ 七、核心设计约束（必须写进文档）

------

## 🔥 约束1：Axis 无分组概念

```text
Axis 不知道自己属于哪个 Group
```

------

## 🔥 约束2：UseCase 不关心 Group

```text
UseCase 不接受 GroupId
```

------

## 🔥 约束3：Group 通过 Context 隔离

```text
不同 Group → 不同 SystemContext
```

------

## 🔥 约束4：Driver 处理所有差异

```text
Axis差异（Y/Z/R） → Driver
PLC差异（Group） → Driver实例
```

------

## 🔥 约束5：UI 不跨层调用

```text
UI → ViewModel → Application → Domain
```
