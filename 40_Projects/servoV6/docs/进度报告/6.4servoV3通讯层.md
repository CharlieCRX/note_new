## MotorCtrl::on_netTgtCmd_rcvd 完整流程分析

### 一、命令传输链路（网络 → ServoAgent → MotorCtrl）

整个命令传输采用 **UDP + JSON** 协议，分以下层次：

#### 1. UDP 网络层 — UdpSrv
- `UdpSrv` 是一个 UDP 服务基类，通过 `QUdpSocket` 在指定 IP:Port 上 `bind()` 监听。
- 收到 UDP 数据包后触发 `readyRead` 信号，调用 `receive()` 虚方法。

#### 2. 协议解析层 — ServoAgent（继承 UdpSrv）
```
receive()                          // 读取UDP数据报，获取发送方地址(host:port)
  → parseMsg(array)                // 入口
    → parseJson(array)             // QJsonDocument 解析JSON → m_lastObj
    → validateFields()             // 校验 "cmd"（命令字）和 "motor"（电机ID）
    → handleCommand()              // 根据m_cmd路由
```

支持的命令枚举：
| cmd值 | 枚举名      | 说明         |
| ----- | ----------- | ------------ |
| 0     | RUN2TARGET  | 绝对位置移动 |
| 1     | RUNOFFSET   | 相对偏移移动 |
| 2     | GETPOSITION | 获取当前位置 |
| 3     | SETSPEED    | 设置速度     |
| 4     | GETSPEED    | 获取速度     |
| 5     | SETORIGIN   | 设置原点     |

#### 3. 命令分发
对 `on_netTgtCmd_rcvd` 的调用有两个入口：

**入口1 — RUN2TARGET（绝对位置移动）**
```cpp
// servoagent.cpp:89-96
```

### 二、on_netTgtCmd_rcvd 内部处理（MotorCtrl.cpp:3555-3576）

```cpp
void MotorCtrl::on_netTgtCmd_rcvd(int motorID, float target)
{
    // Step 1: 测试通讯连接（最多重试3次）
    for (int i = 0; i < 3; i++) {
        if (MotorRegs::get_instance()->testConnection(portName)) break;
    }

    // Step 2: 计算相对偏移量
    // 当前绝对位置 = getLocation(motorID) * 方向系数
    float absPositon = getLocation(motorID) * DbCtrl::m_servoMotor_tb.at(motorID).dir;
    // 相对零点位置（用户设定的零点）
    float relativeOriginPosition = DbCtrl::m_servoMotor_tb.at(motorID).relativeZeroPosition;
    // 当前相对位置 = 绝对位置 - 零点位置
    float relativePosition = absPositon - relativeOriginPosition;
    // 需要移动的偏移量 = 目标绝对位置 - 当前相对位置
    float offset = target - relativePosition;

    // Step 3: 转调 on_netOffCmd_rcvd 执行相对移动
    on_netOffCmd_rcvd(motorID, offset);
}
```

**核心逻辑：**
- 将**绝对目标位置**（target）转换为**相对偏移量**（offset）
- 公式：`offset = target - (current_abs_position - relative_zero_position)`
- 然后调用 `on_netOffCmd_rcvd` 执行相对移动

### 三、on_netOffCmd_rcvd 执行移动（MotorCtrl.cpp:3583-3601）

```cpp
void MotorCtrl::on_netOffCmd_rcvd(int motorID, float offset) {
    // 1. 显示"网络程控"标识
    m_user->setText("网络程控");
    
    // 2. 取消点动模式（使用位置移动模式）
    m_runMode_chb->setChecked(false);  
    
    // 3. 将Modbus协议ID转换为电机功能ID
    motorID = GroupManager::getMotorFuncFromModbusID(motorID);
    
    // 4. 根据电机类型路由到对应的移动函数
    switch (motorID) {
    case M_Z1:    moveZ1(offset);       break;  // Z轴上升/下降
    case M_Z2:    moveZ2(offset);       break;  // Z轴2上升/下降
    case M_Y1:    moveY1(offset);       break;  // Y轴左/右
    case M_Angle: rotate(offset);       break;  // 旋转轴
    }
}
```

各移动函数（如 `moveY1`, `moveZ1`, `rotate`）的共同模式：
1. 设置 `m_isNetCtrl = true` — 标记为网络控制
2. `setClickDist(motorID, abs(offset))` — 设置移动距离
3. 选中对应电机（`selectMotor` 或模拟按钮点击）
4. 点击方向按钮触发移动（`clickDirectionButton` 或直接释放按钮开始点动）

```cpp
void ServoAgent::on_allWorkDone(bool suc) {
    if(m_needRsp) {
        // 获取移动后的当前位置
        float val = 0;
        MotorCtrl::get_instance()->netGetLocation(m_motorID, val);
        // 发送回复（UDP JSON报文）
        sendResponse(suc, nullptr, val);
        m_needRsp = false;  // 重置回复标记
    }
}
```

#### 4.4 sendResponse 构建回复JSON（servoagent.cpp:148-188）
```cpp
void ServoAgent::sendResponse(bool result, const QString &msg, float val) {
    // 将结果写入JSON
    m_lastObj.insert("result", result ? 1 : 0);
    if (!msg.isEmpty()) m_lastObj.insert("msg", msg);
    
    // 不同命令附带不同数据
    switch (cmd) {
    case RUN2TARGET:
    case RUNOFFSET:
        m_lastObj.insert("curr", val);  // 附带当前位置
        break;
    case SETSPEED:
    case SETORIGIN:
        break;  // 仅返回成功/失败
    case GETPOSITION:
        if(result) m_lastObj.insert("curr", val);
        break;
    case GETSPEED:
        if(result) m_lastObj.insert("speed", val);
        break;
    }
    
    // 通过UDP发送回请求端
    send(m_targetHost, m_targetPort, data);
}
```

### 六、关键设计要点

1. **命令缓存**：`ServoAgent::m_needRsp` 标记是否需要发送回复，确保只在有请求时才回复。

2. **绝对←→相对转换**：`on_netTgtCmd_rcvd` 将网络传来的**绝对目标位置**，通过当前读回的位置和相对零点，计算为**相对偏移量**再下发执行。

3. **设置原点复用**：`handleSetOrigin()` 也调用 `on_netTgtCmd_rcvd(motorID, 0)`，即绝对目标位置为0，移动到零点。

4. **电机ID转换**：先通过 `GroupManager::getModbusID()` 将功能ID转为Modbus协议ID，再通过 `GroupManager::getMotorFuncFromModbusID()` 转回功能ID，实现逻辑隔离。

5. **多传输模式兼容**：Pad模式下使用触摸事件（touchBegin/touchEnd），大屏模式使用按钮事件（pressed/released），网络控制与本地控制统一通过 `moveY1`、`moveZ1` 等接口执行。





## UDP 命令格式（JSON over UDP）

### 一、接收命令格式（客户端 → 本系统）

所有命令均为 UDP JSON 报文，**必填字段**：`cmd`（int）、`motor`（int）

电机 ID 映射（`E_MOTOR_ID` 枚举）：
| 功能电机ID | 名称            |
| ---------- | --------------- |
| 0          | M_X1（前后左）  |
| 1          | M_X2（前后右）  |
| 2          | M_Y1（左右）    |
| 3          | M_Z1（上下1）   |
| 4          | M_Z2（上下2）   |
| 5          | M_Angle（旋转） |

#### 1. RUN2TARGET — 绝对位置移动（cmd=0）

请求：
```json
{
    "cmd": 0,
    "motor": 3,
    "target": 500.0
}
```

说明：将电机移动到相对零点偏移 `target` mm 的位置。  
计算逻辑：`offset = target - (当前绝对位置 - 相对零点位置)`，转为相对移动后执行。

---

#### 2. RUNOFFSET — 相对偏移移动（cmd=1）

请求：
```json
{
    "cmd": 1,
    "motor": 3,
    "offset": -100.0
}
```

说明：让电机在当前基础上移动 `offset` mm（正=前进/上/右，负=后退/下/左）。  
直接调用 `on_netOffCmd_rcvd(motorID, offset)`。

---

#### 3. GETPOSITION — 获取当前位置（cmd=2）

请求：
```json
{
    "cmd": 2,
    "motor": 3
}
```

---

#### 4. SETSPEED — 设置速度（cmd=3）

请求：
```json
{
    "cmd": 3,
    "motor": 3,
    "speed": 1500.0
}
```

说明：设置电机运行转速（单位 RPM）。

---

#### 5. GETSPEED — 获取速度（cmd=4）

请求：
```json
{
    "cmd": 4,
    "motor": 3
}
```

---

#### 6. SETORIGIN — 设置原点（cmd=5）

请求：
```json
{
    "cmd": 5,
    "motor": 3
}
```

说明：将电机当前位置设为相对零点。  
内部复用 `on_netTgtCmd_rcvd(motorID, 0)`，即目标位置=0。

---

### 二、回复格式（本系统 → 客户端）

所有回复均包含**原始请求的所有字段**，再加上以下字段：

| 字段     | 类型   | 说明             |
| -------- | ------ | ---------------- |
| `result` | int    | 1=成功，0=失败   |
| `msg`    | string | （可选）错误描述 |

根据不同命令附加额外数据：

#### RUN2TARGET（cmd=0）回复
```json
{
    "cmd": 0,
    "motor": 3,
    "target": 500.0,
    "result": 1,
    "curr": 498.2
}
```
`curr` = 移动完成后的当前位置（mm / °）

#### RUNOFFSET（cmd=1）回复
```json
{
    "cmd": 1,
    "motor": 3,
    "offset": -100.0,
    "result": 1,
    "curr": 398.2
}
```
`curr` = 移动完成后的当前位置

#### GETPOSITION（cmd=2）回复
```json
{
    "cmd": 2,
    "motor": 3,
    "result": 1,
    "curr": 498.2
}
```
`curr` = 当前位置（成功时附带）

#### SETSPEED（cmd=3）回复
```json
{
    "cmd": 3,
    "motor": 3,
    "speed": 1500.0,
    "result": 1
}
```
只有 `result`，无额外数据字段。

#### GETSPEED（cmd=4）回复
```json
{
    "cmd": 4,
    "motor": 3,
    "result": 1,
    "speed": 1500.0
}
```
`speed` = 当前速度 RPM（成功时附带）

#### SETORIGIN（cmd=5）回复
```json
{
    "cmd": 5,
    "motor": 3,
    "result": 1
}
```
只有 `result`，无额外数据字段。

#### 失败回复（通用）
```json
{
    "cmd": 0,
    "motor": 3,
    "target": 500.0,
    "result": 0,
    "msg": "错误描述信息"
}
```
