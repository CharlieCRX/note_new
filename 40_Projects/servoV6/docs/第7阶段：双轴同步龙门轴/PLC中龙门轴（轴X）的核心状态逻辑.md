------

### 一、 联动与基础状态寄存器

这部分寄存器主要用于标识 X1 和 X2 两个物理轴是否成功绑定为逻辑轴 X，以及它们的整体运行状态。

#### 1. `轴X联动状态` (同步联动指示)

- **含义：** 标识龙门轴当前是否处于真正的物理与逻辑联动状态。
- **开启逻辑 (ON)：** 必须同时满足以下条件才会触发联动使能：
  - X1 和 X2 电机的当前速度都必须为 0 (`轴X1CurSpeed = 0.0` 且 `轴X2CurSpeed = 0.0`) 。  
  - X1 和 X2 伺服均已使能 (`轴X1Status AND 轴X2Status`) 。  
  - 收到 `轴X联动使能` 指令，并且两轴当前位置绝对值之和小于 0.1 (`ABS(轴X1CurPosition + 轴X2CurPosition) < 0.1`) 。  
- **关闭逻辑 (OFF)：** 当收到解除联动指令 (`NOT 轴X联动使能`) 时，解除关联 。  

#### 2. `轴X状态显示` (整体状态枚举)

- **含义：** 这是一个枚举寄存器，直接反映龙门轴当前的宏观状态，主要分为 4 个级别 ：  
  - **状态 0 (未使能/停止)：** X1 或 X2 任意一轴未使能 (`NOT 轴X1Status OR NOT 轴X2Status`) 。  
  - **状态 1 (空闲)：** 双轴已使能，但当前没有任何运动 (`轴X1CurSpeed` 和 `轴X2CurSpeed` 均为 0.0) 。  
  - **状态 2 (运动中)：** X1 或 X2 任意一轴正在运动 (速度不为 0.0) 。  
  - **状态 3 (报警/超限位)：** 发生任何指令错误、伺服错误、超差时间到，或者触碰到了硬件限位 (如 `bnslimit`, `bpslimit`) 。

### 二、 运动执行状态寄存器

这部分寄存器用于反馈当前龙门轴正在执行哪种类型的运动，或者运动是否已经结束。前提条件通常是双轴均处于使能状态 (`轴X1Status AND 轴X2Status`) 。  



#### 1. `轴X运动完成`

- **含义：** 标识龙门轴的运动指令是否已经执行结束。
- **逻辑：** 当 X1 和 X2 的所有运动触发信号（绝对定位 MoveDo、相对定位 MoveXDDo、正反点动 JogBackward/JogForward）均 **不处于** 激活状态时，判定为 True (完成) 。否则为 False 。  

#### 2. `轴X绝对定位中` / `轴X相对定位中` / `轴X点动中`

- **含义：** 分别标识当前正在执行的运动模式。
- **逻辑：**
  - **绝对定位中：** X1 或 X2 的绝对定位触发信号 (`MoveDo`) 为 True 。  
  - **相对定位中：** X1 或 X2 的相对定位触发信号 (`MoveXDDo`) 为 True 。  
  - **点动中：** X1 或 X2 的正向或反向点动信号 (`JogBackward` 或 `JogForward`) 为 True 。

### 三、 异常与安全防御寄存器

龙门控制的危险性较高，PLC 在底层做了非常严格的安全熔断逻辑。

#### 1. `轴X软件限位状态`

- **含义：** 龙门轴在联动状态下超出了软件设定的安全移动范围。
- **逻辑：** 当龙门处于联动状态 (`轴X联动状态` 为 True)，并且主参考轴 X1 的当前位置小于等于负限位，或大于等于正限位时触发 。  
- **保护动作：** 触发后，会立刻强制复位绝对定位和相对定位的执行信号 (`轴X1MoveDo:= FALSE`, `轴X1MoveXDDo := false`) 。点动操作也会在对应方向上被拦截 。  

#### 2. `X轴超差标志` 与 `X轴急停触发`

- **含义：** 监控 X1 和 X2 物理横杆是否发生歪曲超差，并执行紧急停机。
- **逻辑：**
  - **超差标志：** 在联动且齿轮比到达状态下，如果两轴位置差的绝对值大于设定的 `轴X超差阈值`，触发超差标志 。  
  - **急停触发：** 超差标志触发后会启动一个定时器。如果持续超差达到设定时间 (`X轴超差时间到`)，则判定为 `X轴急停触发 := TRUE` 。  
- **保护动作：** 发生急停触发时，PLC 会强制清除所有的定位触发、点动触发、回零触发信号，实施紧急制动 。  

#### 3. `轴X报警代码` (详细故障追溯)

- **含义：** 当整体状态 `轴X状态显示` 变为 3 (报警) 时，此寄存器会输出具体的错误原因代码 。  
- **代码定义：**
  - `1`：指令执行报警 (X1 或 X2 指令错误) 。  
  - `2`：伺服报警 (X1 或 X2 伺服反馈了非 0 的错误码) 。  
  - `3`：硬件限位报警 (X1 或 X2 触及物理限位或 DI 状态异常)。**注意：此报警会强制断开轴联动使能 (`轴X联动使能:= FALSE`)** 。  
  - `4`：超差报警 (两轴位置超差且超时) 。  
  - `0`：无报警 。

### 四、 位置数据寄存器

用于向上层实时反馈龙门轴（以 X1 为基准）的空间坐标位置。

- **`轴X当前绝对位置`：** 直接映射物理主轴 `轴X1CurPosition` 的机械绝对坐标 。  
- **`轴X当前相对零点`：** 记录了通过指令设置的相对原点坐标数值 (`轴相对原点记录`) 。  
- **`轴X当前相对位置`：** 计算得出的相对坐标。逻辑为：当前绝对位置减去相对原点记录 (`轴X1CurPosition - 轴相对原点记录`) 。
