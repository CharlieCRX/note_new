## 基于文档「统一命令总线与反馈分发」的具体调整方案

### 一、当前架构问题总结

通过阅读全部源码，确认当前存在以下问题：

| #    | 问题                                                         | 位置                                 |
| ---- | ------------------------------------------------------------ | ------------------------------------ |
| 1    | `ISystemDriver` 继承 `IAxisDriver`，形成"胖接口补丁"：`send()` + `sendGantry()` | `ISystemDriver.h`                    |
| 2    | 缺少 `ServoPowerCommand` 的驱动发送路径 —— `GantryPowerController` 产生命令，但没有地方消费 | `GantryPowerController.h` → 无消费端 |
| 3    | `FakeAxisDriver::sendGantry` 参数类型写的是 `GantryCommand`，但接口声明是 `GantryCouplingCommand`（类型名不一致） | `FakeAxisDriver.h:42`                |
| 4    | `SystemContext` 持有 `ISystemDriver*`，耦合了具体的派生接口  | `SystemContext.h`                    |
| 5    | `GantryOrchestrator` 直接调用 `drv->sendGantry()`，按聚合区分接口 | `GantryOrchestrator.h`               |

---

### 二、修改步骤（共 8 个文件）

#### 步骤 1：定义统一 `SystemCommand`（新建文件）

**新建 `domain/command/SystemCommand.h`**：

```cpp
#pragma once
#include "entity/Axis.h"                              // AxisCommand
#include "gantry/GantryCouplingController.h"          // GantryCouplingCommand
#include "gantry/GantryPowerController.h"             // ServoPowerCommand
#include <variant>

using SystemCommand = std::variant<
    AxisCommand,
    GantryCouplingCommand,
    ServoPowerCommand
>;
```

> **说明**：将三个独立的 Command 统一为一个 variant，作为领域层与基础设施层之间的**唯一命令边界**。

---

#### 步骤 2：重构 `ISystemDriver`（扁平化，不再继承 `IAxisDriver`）

**修改 `infrastructure/ISystemDriver.h`**：

```
当前：
  class ISystemDriver : public IAxisDriver {
      virtual void send(AxisId, const AxisCommand&) = 0;   // 继承自 IAxisDriver
      virtual void sendGantry(const GantryCouplingCommand&) = 0;
  };

修改为：
  class ISystemDriver {
  public:
      virtual ~ISystemDriver() = default;
      virtual void send(const SystemCommand& cmd) = 0;
  };
```

> **关键变化**：
> - 不再继承 `IAxisDriver`
> - `send(AxisId, ...)` 中 `AxisId` 信息**移入 `AxisCommand`**，但这会破坏现有 AxisCommand 结构。替代方案见下方「设计决策待确认」。

---

#### 步骤 3：为 `AxisCommand` 补充 `AxisId` 载体

**问题**：当前 `IAxisDriver::send(AxisId id, const AxisCommand& cmd)` 的 `AxisId` 是独立参数，统一到 `SystemCommand` 后需要解决"这个命令发给哪个轴"的问题。

**两种方案**（请选择）：

| 方案  | 做法                                                         | 优点                  | 缺点                                   |
| ----- | ------------------------------------------------------------ | --------------------- | -------------------------------------- |
| **A** | 保持 `send(SystemCommand)` 签名，但在 `SystemCommand` variant 中增加一个带 `AxisId` 的包装结构 `AxisCommandWithId` | 对 UseCase 层改动最小 | 多一层包装                             |
| **B** | `send(SystemCommand)` 签名不变，由上层调用者在构造 `AxisCommand` 时附加上下文（在 UseCase 层传入 `AxisId`，Driver 内部通过其他方式路由） | 接口纯粹              | 需要在 Driver 实现中额外维护轴路由映射 |

**推荐方案 A**：新建 `struct AxisCommandWithId { AxisId id; AxisCommand cmd; };`，放入 `SystemCommand` variant。

具体修改：

**修改 `domain/command/SystemCommand.h`**，增加：
```cpp
struct AxisCommandWithId {
    AxisId id;
    AxisCommand cmd;
};
```

`SystemCommand` 变为：
```cpp
using SystemCommand = std::variant<
    AxisCommandWithId,
    GantryCouplingCommand,
    ServoPowerCommand
>;
```

---

#### 步骤 4：修改 `IAxisDriver`（标记废弃或直接移除）

**修改 `application/axis/IAxisDriver.h`**：

- **方案一（推荐——更干净）**：直接删除此文件，所有引用替换为 `ISystemDriver`
- **方案二（保守）**：保留但标记 `[[deprecated]]`，转发到统一接口

推荐方案一。影响范围：
- `EnableUseCase.h` 中 `drv->send(axisId, cmd)` → `drv->send(AxisCommandWithId{axisId, cmd})`
- `MoveAbsoluteUseCase.h` 同上
- `MoveRelativeUseCase.h` 同上
- `JogAxisUseCase.h` 同上
- `StopAxisUseCase.h` 同上
- `AutoAbsMoveOrchestrator.h` 同上
- `AutoRelMoveOrchestrator.h` 同上
- `JogOrchestrator.h` 同上
- `SystemContext.h` 中 `ISystemDriver*` 保持不变（但接口内方法签名变了）

---

#### 步骤 5：修改 `FakeAxisDriver` 适配新接口

**修改 `infrastructure/FakeAxisDriver.h`**：

```
当前：
  void send(AxisId id, const AxisCommand& cmd) override { ... }
  void sendGantry(const GantryCommand& /*cmd*/) override { ... }  // ← 类型名还有 bug

修改为：
  void send(const SystemCommand& cmd) override {
      std::visit([this](auto&& c) {
          handle(c);
      }, cmd);
  }

  // 新增三个 handle 重载
  void handle(const AxisCommandWithId& c) {
      history.push_back({c.id, c.cmd});
      m_plc.onCommand(c.id, c.cmd);
  }
  void handle(const GantryCouplingCommand& c) { /* PLC 龙门处理 */ }
  void handle(const ServoPowerCommand& c)    { /* PLC 电机使能处理 */ }
```

---

#### 步骤 6：修改 `GantryOrchestrator` 调用方式

**修改 `application/policy/GantryOrchestrator.h`**：

```
当前：

修改为：
  drv->send(cmd);   // GantryCouplingCommand 直接作为 SystemCommand 发送
```

#### 步骤 7：修改所有 UseCase 调用方式

**影响文件**（5 个 UseCase + 4 个 Orchestrator）：

每个文件中的：
```cpp

```

改为：
```cpp
drv->send(AxisCommandWithId{axisId, axis->getPendingCommand()});
```

#### 步骤 8：修改 `FakePLC` 扩展龙门指令处理

**修改 `infrastructure/FakePLC.h`**：

当前 `FakePLC` 只处理 `AxisCommand`，需要增加 `GantryCouplingCommand` 和 `ServoPowerCommand` 的处理（或在 `FakeAxisDriver` 层做转换）。

推荐在 `FakePLC` 中新增：
```cpp
void onGantryCommand(const GantryCouplingCommand& cmd) { ... }
void onServoPowerCommand(const ServoPowerCommand& cmd) { ... }
```
