## Step 1：创建 `domain/gantry/GantryFeedback.h`（独立 DTO）

**操作**：新建文件

**内容**：从 `GantryGroup.h` 将 `GantryFeedback` 结构体抽出，并新增 `enable` 字段。

```cpp
// domain/gantry/GantryFeedback.h  (新文件)
#pragma once

struct GantryFeedback {
    bool enable;        // PLC 寄存器「轴X状态显示」：1 = 电机使能, 0 = 掉电
    bool isCoupled;     // PLC 寄存器「轴X联动状态」：ON/OFF
    int  errorCode;     // PLC 寄存器「Gantry_Error_Code」
};
```

**理由**：
- `GantryFeedback` 是独立 DTO，用于在 PLC 轮询线程和领域对象之间传输数据
- 新增 `enable` 字段使反馈完整覆盖三个龙门相关 PLC 寄存器
- 抽离为独立文件后，`GantryMotorController` 和 `GantryGroup` 可共享同一个头文件，避免循环依赖

## Step 3：创建 `domain/gantry/GantryMotorController.h`（新增领域类）

**操作**：新建文件

### 3.1 接口设计

```cpp
// domain/gantry/GantryMotorController.h  (新文件)
#pragma once

#include "gantry/GantryFeedback.h"
#include "gantry/GantryRejection.h"
#include <optional>

/**
 * @brief 龙门电机控制器
 *
 * 职责：控制 PLC 寄存器「使能轴X电机」（同时使能 X1/X2 电机）。
 *      在任何联动状态下均可访问（不经过 SystemContext 的龙门锁定逻辑）。
 *
 * PLC 硬件约束：
 *  - 「使能轴X电机」寄存器同时控制 X1/X2 电机
 *  - 「轴X状态显示」寄存器反馈电机是否已使能
 */
struct MotorCommand {
    bool enable;  // true = 使能, false = 掉电
};

class GantryMotorController {
public:
    // ========== 意图生成 ==========

    /**
     * @brief 请求使能/掉电龙门电机
     * @return None           成功（包括幂等：已在目标状态）
     *         NotSynchronized 尚未收到第一帧 PLC 反馈，拒绝操作
     */
    GantryRejection requestEnable(bool active) {
        if (!m_synchronized) return GantryRejection::NotSynchronized;
        if (m_enabled == active) return GantryRejection::None;  // 幂等
        m_enabled = active;
        m_pending_command = MotorCommand{active};
        return GantryRejection::None;
    }

    // ========== 状态查询 ==========

    bool isEnabled() const { return m_enabled; }
    bool isSynchronized() const { return m_synchronized; }
    bool isNotSynchronized() const { return !m_synchronized; }

    // ========== 命令弹出 ==========

    bool hasPendingCommand() const { return m_pending_command.has_value(); }

    MotorCommand popPendingCommand() {
        auto cmd = *m_pending_command;
        m_pending_command.reset();
        return cmd;
    }

    // ========== 反馈接收（由 PLC 轮询线程调用） ==========

    /**
     * @brief 接收 PLC 寄存器「轴X状态显示」的反馈
     *        首次反馈自动标记 m_synchronized = true（退出未同步态）
     */
    void applyFeedback(const GantryFeedback& feedback) {
        m_synchronized = true;
        m_enabled = feedback.enable;
    }

private:
    bool m_enabled = false;
    bool m_synchronized = false;
    std::optional<MotorCommand> m_pending_command;
};
```

### 3.2 设计决策

| 决策                                                         | 理由                                                 |
| ------------------------------------------------------------ | ---------------------------------------------------- |
| `m_synchronized` 独立于 `GantryCouplingState::NotSynchronized` | 电机控制器只需知道是否收到过反馈，不需要五态状态机   |
| `requestEnable` 在任何联动状态下均可调用                     | 使能电机不依赖联动状态，由调用方自己保证语义正确性   |
| 幂等内建                                                     | `m_enabled == active` → 直接返回 None，不生成新命令  |
| 消费 `GantryFeedback::enable`                                | PLC 轮询线程将同一份 `GantryFeedback` 推送给两个对象 |

## Step 6：修改 `application/policy/GantryOrchestrator.h`（核心悖论修复）

**操作**：编辑现有文件

### 6.1 移除 `tryGetAxis(AxisId::X)` 调用

**旧代码（悖论根源）**：
```cpp
void tick() {
    SystemContext* group = nullptr;
    ContextRejection mgrReason = ContextRejection::None;
    if (!m_manager.tryGetGroup(m_groupName, group, mgrReason)) {
        m_step = Step::Error;
        m_lastError = mgrReason;
        return;
    }

    Axis* x = nullptr;                              // ──┐
    ContextRejection ctxReason = ContextRejection::None;  //   ├── 悖论根源
    if (!group->tryGetAxis(AxisId::X, x, ctxReason)) {     //   │   Decoupled 时 X 被拒绝
        m_step = Step::Error;                              //   │
        m_lastError = ctxReason;                           // ──┘
        return;
    }

    GantryGroup& gantry = group->gantry();
    // ...
}
```

**新代码（修复后）**：
```cpp
void tick() {
    SystemContext* group = nullptr;
    ContextRejection mgrReason = ContextRejection::None;
    if (!m_manager.tryGetGroup(m_groupName, group, mgrReason)) {
        m_step = Step::Error;
        m_lastError = mgrReason;
        return;
    }

    // 不再通过 tryGetAxis(AxisId::X) 获取轴指针
    // 龙门流程仅需要以下两个领域对象：
    GantryGroup& gantry = group->gantry();
    GantryMotorController& motor = group->gantryMotor();

    switch (m_step) {
        // ...
    }
}
```

### 6.2 修改 `EnsuringEnabled` 步骤

**旧**：
```cpp
case Step::EnsuringEnabled: {
    auto err = EnableUseCase{}.execute(m_manager, m_groupName, AxisId::X, true);
    if (std::holds_alternative<std::monostate>(err)) {
        m_step = Step::WaitingEnabled;
    } else {
        m_step = Step::Error;
        m_lastError = err;
    }
    break;
}
```

**新**：
```cpp
case Step::EnsuringEnabled: {
    // 直接通过电机控制器下发使能命令
    // 不再经过 EnableUseCase（它依赖 tryGetAxis(AxisId::X)，会在解耦状态下被拒绝）
    auto result = motor.requestEnable(true);
    if (result == GantryRejection::None) {
        // 若有 pending 命令，下发到驱动
        if (motor.hasPendingCommand()) {
            if (auto* drv = group->driver()) {
                auto cmd = motor.popPendingCommand();
                drv->sendGantryMotor(cmd);  // 见 Step 7
            }
        }
        m_step = Step::WaitingEnabled;
    } else {
        m_step = Step::Error;
        m_lastError = result;
    }
    break;
}
```

### 6.3 修改 `WaitingEnabled` 步骤

**旧**：
```cpp
case Step::WaitingEnabled:
    if (x->state() == AxisState::Idle) {
        m_step = Step::Coupling;
    } else if (x->state() == AxisState::Error) {
        m_step = Step::Error;
        m_lastError = x->lastRejection();
    }
    break;
```

**新**：
```cpp
case Step::WaitingEnabled:
    if (motor.isEnabled()) {
        m_step = Step::Coupling;
    }
    // 电机控制器不暴露错误状态——PLC 的「轴X状态显示」只有 0/1
    // 电机使能失败 = 超时（可选实现），暂保持等待
    break;
```

### 6.4 其他步骤不变

`Coupling`、`WaitingCoupled`、`Decoupling`、`WaitingDecoupled`、`Done`、`Error` 步骤保持不变。

## Step 8：更新 `infrastructure/FakePLC.h`（新增龙门反馈生成）

**操作**：编辑现有文件

### 8.1 新增龙门反馈方法

```cpp
/**
 * @brief 生成龙门反馈快照（PLC 轮询线程视角）
 *
 * 读取：
 *  - Axis::X1/Axis::X2 电机状态 → 推导 enable
 *  - 龙门联动状态内部寄存器 → isCoupled / errorCode
 */
GantryFeedback getGantryFeedback() const {
    // 「轴X状态显示」：X1/X2 任一使能即视为电机已使能
    bool x1Enabled = (m_axes.at(AxisId::X1).feedback.state != AxisState::Disabled);
    bool x2Enabled = (m_axes.at(AxisId::X2).feedback.state != AxisState::Disabled);

    // 「轴X联动状态」：目前 FakePLC 没有独立联动寄存器，
    // 初期简化为总是返回 isCoupled=false, errorCode=0
    // 后续可扩展为完整的龙门联动仿真
    return GantryFeedback{
        .enable    = (x1Enabled && x2Enabled),
        .isCoupled = false,
        .errorCode = 0
    };
}
```

**注意**：`GantryFeedback::enable` 反映 X1/X2 电机**都已**使能。如需要更精细的反馈（单轴使能状态），后续扩展。

```cpp
    motor->applyFeedback({ .enable = false, .isCoupled = false, .errorCode = 0 });
```

### 12.3 新增悖论验证测试 ⭐

```cpp
// ============================================================
// 悖论验证：确保 Decoupled 状态下联动流程不被死锁
// ============================================================

// 10. 悖论验证：Decoupled 状态下 startCoupling 能推进到 EnsuringEnabled
//     不会因为 tryGetAxis(AxisId::X) 返回 LogicalAxisUnavailableWhenDecoupled 而失败
TEST_F(GantryOrchestratorTest, coupling_does_not_deadlock_when_decoupled)
{
    // Given: 已同步但解耦（电机未使能）
    motor->applyFeedback({ .enable = false, .isCoupled = false, .errorCode = 0 });
    gantry->applyFeedback({ .enable = false, .isCoupled = false, .errorCode = 0 });

    orchestrator->startCoupling();

    // tick 不应该立即进入 Error（修复前会因 tryGetAxis 失败而入 Error）
    orchestrator->tick();

    EXPECT_NE(orchestrator->currentStep(), GantryOrchestrator::Step::Error);
    EXPECT_EQ(orchestrator->currentStep(), GantryOrchestrator::Step::WaitingEnabled);
}

// 11. 悖论验证：NotSynchronized 状态下 startCoupling 立即进入 Error
TEST_F(GantryOrchestratorTest, coupling_fails_when_not_synchronized)
{
    // Given: not synchronized（电机控制器和龙门状态机均未收到反馈）
    // motor.applyFeedback() 和 gantry.applyFeedback() 均未调用

    orchestrator->startCoupling();
    orchestrator->tick();

    EXPECT_EQ(orchestrator->currentStep(), GantryOrchestrator::Step::Error);
    EXPECT_TRUE(std::holds_alternative<GantryRejection>(orchestrator->lastError()));
    EXPECT_EQ(std::get<GantryRejection>(orchestrator->lastError()), GantryRejection::NotSynchronized);
}

// 12. 悖论验证：Decoupled 状态下完整联动流程（电机使能 + 联动）成功到达 Done
TEST_F(GantryOrchestratorTest, full_coupling_flow_from_decoupled)
{
    // Given: 已同步，解耦，电机未使能
    motor->applyFeedback({ .enable = false, .isCoupled = false, .errorCode = 0 });
    gantry->applyFeedback({ .enable = false, .isCoupled = false, .errorCode = 0 });

    orchestrator->startCoupling();

    // EnsuringEnabled → motor.requestEnable(true)
    orchestrator->tick();
    ASSERT_EQ(orchestrator->currentStep(), GantryOrchestrator::Step::WaitingEnabled);

    // 模拟 PLC 反馈：电机已使能
    motor->applyFeedback({ .enable = true, .isCoupled = false, .errorCode = 0 });
    orchestrator->tick();
    ASSERT_EQ(orchestrator->currentStep(), GantryOrchestrator::Step::Coupling);

    // Coupling → gantry.requestCouple(true)
    orchestrator->tick();
    ASSERT_EQ(orchestrator->currentStep(), GantryOrchestrator::Step::WaitingCoupled);

    // 模拟 PLC 反馈：联动成功
    gantry->applyFeedback({ .enable = true, .isCoupled = true, .errorCode = 0 });
    orchestrator->tick();
    ASSERT_EQ(orchestrator->currentStep(), GantryOrchestrator::Step::Done);

    EXPECT_TRUE(orchestrator->isDone());
    EXPECT_TRUE(gantry->isCoupled());
}
```
