# Gantry 联动状态机重构说明（基于 Clean Architecture + TDD）

你当前已经完成了：

```text

→ WaitingAxisReady
→ Couple
→ WaitingCoupled
```

的状态机框架。

这是一个非常好的开始。

但是随着：

```text

```


因为：

```text
系统开始进入：

```

模型。

# 一、当前存在的问题（核心）

当前：

```cpp
m_gantry.couple();
```

内部：

```cpp

```

会：

```text
立即进入：
```

导致：

```cpp

```

下一 tick：



# 三、但真实 PLC 不是这样的

PLC 的真实业务规则是：

```text

≠

```

PLC 内部还会判断：


```text

```

# 四、这意味着什么？

意味着：

# 当前系统：

```text

vs
```

分离。

必须区分：

| 类型     | 含义          |
| -------- | ------------- |
| Intent   | 我请求联动    |

------

# 十六、因此：

# GantryCouplingState

建议修改为：

```cpp
class GantryCouplingState {
public:
    enum class Status {
        Decoupled,
        CouplingRequested,
        Coupled
    };

    Status status() const
    {
        return m_status;
    }

    bool isCoupled() const
    {
        return m_status == Status::Coupled;
    }

    bool isCouplingRequested() const
    {
        return m_status == Status::CouplingRequested;
    }

    void requestCouple()
    {
        m_status = Status::CouplingRequested;
    }

    void markCoupled()
    {
        m_status = Status::Coupled;
    }

    void decouple()
    {
        m_status = Status::Decoupled;
    }

private:
    Status m_status = Status::Decoupled;
};
```
