# 🧠 一、龙门轴的“统一语义模型”

先统一一个认知（非常关键）：

```text
X（龙门轴）是唯一对外轴
X1 / X2 是实现细节（物理执行单元）
```

------

## 🎯 统一语义定义

```text
方向：
    Forward  = 远离操作者
    Backward = 靠近操作者

位置：
    X.position = 龙门整体位置（统一坐标系）

运动：
    所有操作都针对 X，而不是 X1 / X2
```

------

# 🧩 二、龙门轴业务语义约束（完整体系）

我帮你整理为 **6大类约束（最终版）**

------

# 🧩 1. 模式语义约束（Mode Semantics）

------

## 约束1：模式定义

```text
龙门轴存在两种模式：

1️⃣ 联动模式（Coupled）
    → X 是唯一可操作对象

2️⃣ 分动模式（Decoupled）
    → X 不可操作
    → 只能操作 X1 或 X2
```

------

## 约束2：模式互斥

```text
任意时刻只能处于一种模式：

Coupled ⊕ Decoupled
```

------

## 约束3：联动模式的语义

```text
在 Coupled 模式下：

所有操作：
    Jog / MoveAbsolute / MoveRelative

语义：
    = 对“X整体”的操作
```

👉 Domain 不关心 X1/X2

------

## 约束4：分动模式的语义

```text
在 Decoupled 模式下：

操作对象：
    X1 或 X2

X（龙门轴）：
    不可被操作（只读状态）
```

------

# 🧩 2. 方向语义约束（Direction Semantics）

------

## 约束5：方向唯一性（核心）

```text
系统中只允许存在一个方向语义：

Forward / Backward（统一定义）
```

------

## 约束6：方向与物理无关

```text
方向定义：

Forward：
    龙门整体远离操作者

Backward：
    龙门整体靠近操作者
```

👉 ❗与电机正反转、顺逆时针无关

------

## 约束7：所有命令必须基于逻辑方向

```text
任何 Jog / Move：

必须使用：
    Forward / Backward

禁止：
    使用物理方向（CW/CCW）
```

------

# 🧩 3. 位置语义约束（Position Semantics）

------

## 约束8：统一位置定义

```text
X.position = 龙门整体位置（逻辑坐标）
```

------

## 约束9：位置一致性约束（核心）

```text
必须满足：

X1.pos ≈ -X2.pos   （镜像关系）
```

------

## 约束10：逻辑位置计算

```text
X.position = X1.pos
```

------

## 约束11：位置一致性失效

```text
如果：

|X1.pos + X2.pos| > epsilon

则：

→ 系统认为“不同步”
→ 禁止进入联动模式
```

------

# 🧩 4. 联动建立约束（Coupling Constraint）

------

## 约束12：联动是“状态申请”

```text
Coupled 模式不是强制切换：

而是：
    “申请进入联动状态”
```

------

## 约束13：联动建立条件（关键）

```text
必须满足：

1. X1 / X2 均已使能
2. 无报警
3. 未触发限位
4. 满足位置一致性：
   |X1.pos + X2.pos| <= epsilon
```

否则：

```text
→ 拒绝进入 Coupled
```

------

## 约束14：联动期间持续约束

```text
在 Coupled 模式运行中：

必须持续满足：

|X1.pos + X2.pos| <= Threshold
```

否则：

```text
→ 触发 DeviationFault
→ 强制退出联动
```

------

# 🧩 5. 安全语义约束（Safety Constraint）

------

## 约束15：限位优先级最高

```text
任何时刻：

一旦触发限位：

→ 所有运动立即视为非法
```

------

## 约束16：限位后的行为限制（关键）

```text
在 Limit 状态下：

只允许：
    ✔ Jog（远离限位方向）

禁止：
    ❌ MoveAbsolute
    ❌ MoveRelative
    ❌ 朝限位方向 Jog
```

------

## 约束17：报警约束

```text
在 Alarm 状态下：

禁止所有运动操作

只允许：
    ResetAlarm
```

------

# 🧩 6. 激活与操作约束（Activation Constraint）

------

## 约束18：操作对象互斥

```text
Coupled 模式：
    只能操作 X

Decoupled 模式：
    只能操作 X1 或 X2

禁止：
    同时操作多个对象
```

------

## 约束19：运动互斥

```text
任意时刻：

只允许存在一个运动意图：

Jog ⊕ MoveAbsolute ⊕ MoveRelative
```

------

## 约束20：状态一致性（聚合语义）

```text
X.state = 聚合状态：

if 任一轴：
    Moving → X = Moving
    Alarm → X = Alarm
    Limit → X = Limit
```

------

# 🧠 三、最终“龙门轴语义真理”（总结）

你可以把下面这段直接写进设计文档👇

------

```text
1. X 是唯一对外暴露的逻辑轴
2. X1/X2 是实现细节，不参与业务决策
3. 所有操作基于统一逻辑方向（Forward/Backward）
4. 联动是状态，不是命令
5. 分动与联动互斥
6. 联动建立必须满足位置一致性
7. 运行过程中必须持续满足同步约束
8. 限位与报警优先级高于一切运动
9. Domain 负责“是否允许操作”，不负责“如何执行”
10. 所有物理差异（方向/坐标）由 HAL 层消化
```
