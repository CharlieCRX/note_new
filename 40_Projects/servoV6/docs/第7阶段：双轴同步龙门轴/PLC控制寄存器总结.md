根据您提供的 PLC 程序文档，该双轴龙门结构（以 X 轴为例，包含 X1 和 X2 两个电机）的控制与状态寄存器配合逻辑非常严谨，主要围绕**主从轴同步（联动）**、**独立控制**以及安全保护（限位与超差）三个核心维度展开。

以下是为您梳理的寄存器与控制逻辑架构：

### 1. 基础使能与急停逻辑 (Enable & E-Stop)

整个龙门系统的底层控制首先受制于设备的全局急停状态，确保在任何危险情况下切断动力和指令。

- **电机使能**：X1 和 X2 轴的物理使能严格依赖于主控的 `使能轴X电机` 信号，并且前提是必须处于 `NOT 设备急停` 状态 。  
- **急停复位指令**：当设备处于急停中时，PLC 会强制将所有的运动触发信号（包括点动正反转、绝对定位、相对定位等）置为 `FALSE`，防止急停解除后发生意外窜动 。  
- **急停连锁清除**：如果触发了 X 轴专属的急停信号（如超差或软限位触发），系统会立即切断 X1 的所有运动执行标志（如 `轴X1MoveDo`、`轴X1HomeDo`、`轴X1ZeroDo` 等）并复位定位触发位 。  

### 2. 龙门联动与解锁机制 (Linkage Synchronization)

龙门结构的核心在于两根平行轴的同步。程序通过特定的寄存器条件来决定 X1 和 X2 是处于“刚性联动”还是“解除关联”状态。

- **建立联动的前提**：只有当 `轴X联动使能` 信号有效，并且 X1 与 X2 当前的绝对位置差值小于 0.1 (`ABS(轴X1CurPosition - 轴X2CurPosition) < 0.1`) 时，系统才会真正将 `X轴关联触发` 置位 。  
- **解除关联**：当 `轴X联动使能` 被撤销（NOT 状态）时，程序会触发 `X轴解除关联触发`，允许两轴独立操作 。  
- **主从速度跟随**：在速度寄存器分配上，程序明确将 X1 设为主轴，X2 设为从轴 。X2 的定位速度 (`轴X2MoveSpeed`) 和手动点动速度 (`轴X2JogSpeed`) 始终被强制等于 X1 的设定速度 。  

### 3. 定位与运动执行逻辑 (Positioning Control)

程序的运动执行严格区分了“联动状态”和“单轴独立状态”，以防止机械结构的扭曲损坏。

- **联动下的协同定位**：当处于联动状态、未解除关联、速度为 0 且无其他运动指令（如点动、回零）时，接收到 `轴X绝对定位触发`，系统才会将目标距离赋予 `X1MovePosition`，并激活 `轴X1MoveDo := TRUE` 执行运动 。  

- **解除联动下的独立定位**：如果在非联动状态（`NOT 轴X联动使能`），X1 和 X2 可以接收各自的独立绝对定位指令（`轴X1单独绝对定位触发` / `轴X2单独绝对定位触发`），各自向独立的坐标点运动，用于机械调平或原点校准 。  

- **点动（Jog）保护**：点动操作不仅受联动状态的约束，还受到软件限位寄存器的严格打断。如果当前位置小于等于负限位，反转信号会被强制置 `FALSE`；大于等于正限位，正转信号被强制置 `FALSE` 。  

  

  

### 4. 超差保护与报警状态机 (Tolerance Protection & Alarms)

对于双驱龙门，两轴位置偏差过大（超差）会导致机械结构拉扯变形，该程序设计了完善的容错与报警体系。

- **超差监控与急停**：在联动定位期间，程序实时监控两轴位置。若 `ABS(轴X1CurPosition + 轴X2CurPosition) > 轴X超差阈值`，则将 `X轴超差标志` 置为 `TRUE` 。  

- **超差滤波计时**：超差标志通过一个 TONR 定时器进行滤波判断（预设值 PT=20）。如果超差持续时间达到阈值，则输出 `X轴超差时间到`，进而触发 `X轴急停触发` 。  

- **状态码映射 (报警代码 `轴X报警代码`)**：

  - **代码 1**：当 X1 或 X2 出现指令错误时触发，表示指令执行报警 。  

  - **代码 2**：当 X1 或 X2 伺服驱动器报错（伺服错误码不为 0）时触发，表示底层硬件伺服报警 。  

  - **代码 3**：当遇到硬限位（bnslimit/bpslimit）或伺服 DI 异常时触发限位报警，并且系统会为了安全**自动强制断开龙门联动** (`轴X联动使能:= FALSE`) 。  

  - **代码 4**：由上述的 `X轴超差时间到` 触发，明确指示发生了机械不同步超差 。
