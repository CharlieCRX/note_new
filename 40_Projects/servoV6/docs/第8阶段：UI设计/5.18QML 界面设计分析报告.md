## QML 界面设计分析报告

### 一、ViewModel 层核心数据与命令接口

**AxisViewModelCore** 是纯逻辑核心，持有：
- `std::unordered_map<AxisId, AxisState>` — 所有轴的实时遥测数据（位置、速度、扭矩、状态）
- `std::vector<ErrorContext>` — 全局错误列表（含时间戳）
- 6 个动作命令：`moveAbsolute`、`moveRelative`、`jogAxis`、`stopAxis`、`enable`、`emergencyStop`、`releaseEmergencyStop`

**QtAxisViewModel** 是 QML 适配层，暴露的关键 Q_PROPERTY：
| 属性              | 类型                 | 用途                           |
| ----------------- | -------------------- | ------------------------------ |
| `axisStates`      | `QVector<AxisState>` | 列表模型数据源，驱动轴列表渲染 |
| `statusMessage`   | `QString`            | 全局状态消息                   |
| `emergencyActive` | `bool`               | 急停状态指示                   |

**ErrorTranslator** 提供错误码 → 人类可读消息的映射。

### 二、数据流向与绑定设计

```
          PLC 硬件数据
               │
    ISystemDriver (infrastructure)
               │
    AxisSyncService → AxisRepository → AxisState 更新
               │
    AxisViewModelCore (纯逻辑，持有 AxisState 集合)
               │
    QtAxisViewModel (QML 适配，QVector<AxisState> + Q_PROPERTY)
               │
    ┌──────────┼──────────┐
    │          │          │
    ▼          ▼          ▼
Repeater   Telemetry   ActionControl
(轴列表)   (遥测展示)   (命令触发)
```

**关键绑定链路**：
1. `Repeater` 的 `model` 绑定到 `QtAxisViewModel.axisStates`，`AxisItemDelegate` 从 `model` 角色中取 `position`、`velocity`、`status` 等字段渲染每一行
2. 选中轴后，`TelemetryBlock` 的 position/velocity/torque Text 绑定到 `modelData.position` 等
3. `ActionControlBlock` 的按钮 `onClicked` 调用 `QtAxisViewModel.moveAbsolute()` 等 Q_INVOKABLE 方法
4. `ErrorPanelBlock` 绑定到 `QtAxisViewModel.statusMessage` 展示错误/状态

### 四、各组件绑定设计详解

#### 4.1 AxisSelectorBlock（轴列表）
```qml
ListView {
    model: viewModel.axisStates   // 直接绑定 QVector<AxisState>
    delegate: AxisItemDelegate {
        text: modelData.axisId
        position: modelData.position
        velocity: modelData.velocity
        status: modelData.status
        hasError: modelData.errorCode != 0
        onClicked: root.selectedAxisId = modelData.axisId
    }
}
```

#### 4.2 TelemetryBlock（遥测详览）
```qml
// 根据 selectedAxisId 从 viewModel 中取对应 AxisState
Text { text: "Position: " + selectedAxis.position }
Text { text: "Velocity: " + selectedAxis.velocity }
Text { text: "Torque: "   + selectedAxis.torque }
Rectangle { color: statusColor(selectedAxis.status) }
```

#### 4.3 ActionControlBlock（操作面板）
```qml
Button { text: "Jog+";  onClicked: viewModel.jogAxis(axisId, +dir, velocity) }
Button { text: "Jog-";  onClicked: viewModel.jogAxis(axisId, -dir, velocity) }
Button { text: "Move Abs"; onClicked: viewModel.moveAbsolute(axisId, target, vel) }
Button { text: "Move Rel"; onClicked: viewModel.moveRelative(axisId, delta, vel) }
Button { text: "Stop";  onClicked: viewModel.stopAxis(axisId) }
Button { text: "Enable"; onClicked: viewModel.enable(axisId) }
IndustrialButton { text: "E-STOP"; onClicked: viewModel.emergencyStop() }
```

#### 4.4 ErrorPanelBlock（错误面板）
```qml
// 绑定全局状态消息
Text { 
    text: viewModel.statusMessage
    color: viewModel.emergencyActive ? "red" : "white"
}
// 可扩展为错误列表：
Repeater {
    model: viewModel.errors  // 如果 QtAxisViewModel 暴露错误列表
    delegate: Text { text: modelData.message }
}
```

### 五、关键交互设计要点

| 交互场景         | 设计方案                                                     |
| ---------------- | ------------------------------------------------------------ |
| **多轴选择**     | AxisSelectorBlock 支持单击选中单轴，ActionControlBlock 操作针对 `selectedAxisId` |
| **Jog 持续运动** | 按钮 `onPressed` 调用 `jogAxis()`，`onReleased` 调用 `stopAxis()` — 实现按住移动、松开停止 |
| **速度参数**     | VelocitySettingsPopup 弹出层设置全局速度值，供 Jog/Move 操作使用 |
| **急停优先级**   | EmergencyStop 按钮始终可用，不依赖轴选择；`emergencyActive` 状态驱动全局 UI 变红 |
| **错误展示**     | ErrorPanelBlock 绑定 `statusMessage`；后期可扩展为带时间戳的错误列表 |
| **遥测刷新**     | `axisStates` 属性通过 NOTIFY 信号自动驱动 QML 绑定更新（PLC 轮询驱动） |

### 七、建议增强

1. **QtAxisViewModel 暴露 `selectedAxisId` 属性** → 简化 QML 侧的选中轴状态管理
2. **错误列表 QVector<ErrorEntry> 暴露** → ErrorPanelBlock 可展示多条历史错误
3. **AxisState 增加 `isMoving` 属性** → UI 可据此显示运动动画/禁止按钮
4. **VelocitySettingsPopup 结果回传** → 通过 signal 将速度值传回 ActionControlBlock
