------

## 🏗️ 第一部分：数据结构与同步约束 (Data Mirroring)

`Axis` 类作为 PLC 寄存器状态的忠实镜像，实现“单一事实来源”。

- **反馈扩展**：`AxisFeedback` 必须包含 `absPos`（绝对位置）、`relPos`（相对位置）以及 **`relZeroAbsPos`（PLC 记录的相对零点所在的绝对位置）**。
- **镜像读取**：
  - `currentRelativePosition()`：直接返回反馈包中的 `relPos`。
  - **`relativeZeroAbsolutePosition()`**：直接返回反馈包中的 `relZeroAbsPos`。
- **同步原子性**：每当执行 `applyFeedback` 时，所有位置寄存器必须同时更新，确保逻辑层获取的坐标系镜像具有时空一致性。

------

## 🛡️ 第二部分：指令准入约束 (Group B: Strict Idle Guard)

确保基准修改安全性，防止在运行或异常态下发生坐标漂移。

- **唯一准入状态**：`setRelativeZero()` 和 `clearRelativeZero()` 仅在 $m\_state == AxisState::Idle$ 时允许执行。
- **全状态拦截**：在 `Moving`（Jog/Move）、`Disabled`、`Unknown` 或 `Error` 状态下，指令必须返回 `false`。
- **捕获快照**：在 `setRelativeZero()` 被接受的瞬间，`Axis` 必须在内部捕获当前的 `absPos` 作为**预期基准值**（Target Base），用于后续的闭环校验。

------

## 🔄 第三部分：指令生命周期与闭环约束 (Wait-for-PLC)

意图的消费执行“双重校验”逻辑，通过数学容差 $\epsilon$ ($0.001$) 判定。

- **设置相对原点 (SetRelativeZero)**：
  - **触发**：调用后 `hasPendingCommand()` 变为 `true`。
  - **闭环条件（双重判定）**：必须同时满足以下两个条件意图才会消失：
    1. **坐标归零**：$|relPos| < \text{POSITION\_EPSILON}$。
    2. **基准对齐**：反馈的 **$|relZeroAbsPos - \text{Target Base}| < \text{POSITION\_EPSILON}$**。
- **清除相对原点 (ClearRelativeZero)**：
  - **触发**：调用后 `hasPendingCommand()` 变为 `true`。
  - **闭环条件（双重判定）**：
    1. **坐标恢复**：$|relPos - absPos| < \text{POSITION\_EPSILON}$。
    2. **基准重置**：反馈的 **$|relZeroAbsPos| < \text{POSITION\_EPSILON}$**（判定 PLC 已清空基准寄存器）。
- **绝对位置清零 (ZeroAbsolute)**：
  - **闭环条件**：$|absPos| < \text{POSITION\_EPSILON}$。

------

## ⚙️ 第四部分：运动定位的完成度约束

- **数值达标判定**：Move 指令的意图清理不仅要求状态回到 `Idle`，还必须满足物理数值闭环。
- **绝对定位闭环**：$|absPos - Target| < \text{POSITION\_EPSILON}$。
- **相对定位闭环**：$|absPos - (StartAbsPos + Distance)| < \text{POSITION\_EPSILON}$。
