针对你提出的软限位需求，我们将业务逻辑抽象为**“三位一体”的安全约束模型**：即**数据镜像（知觉）**、**边界预检（预判）\**与\**状态锁死（防御）**。

这种抽象不仅能满足 HMI 展示的需求，更能确保上位机与 PLC 在物理边界上达成逻辑共识。

------

## 🏗️ 软限位（Soft Limits）业务约束抽象文档

### 第一部分：状态与配置镜像 (State & Config Mirroring)

`Axis` 必须实时同步 PLC 的边界状态与配置参数，确保 HMI 展示与逻辑校验的“单一事实来源”。

- **双态镜像 (Status Bits)**：实时镜像 PLC 反馈的“正向限位触发”与“负向限位触发”标志位。
- **数值镜像 (Limit Values)**：从 PLC 读取当前生效的软限位坐标值（$PosLimitValue$, $NegLimitValue$），用于界面展示及上位机预校验。
- **同步原子性**：限位状态位与限位数值必须在同一反馈包中更新，防止因坐标系重标定（如清零）导致的数值与状态不匹配。

### 第二部分：静态准入预检 (Pre-flight Validation)

在指令产生意图并存入 `m_pending_intent` 之前，必须进行数值边界检查。

- **绝对定位拦截**：目标位置 $P_{target}$ 必须满足：

  $$NegLimitValue \le P_{target} \le PosLimitValue$$

  若超出此范围，`moveAbsolute` 必须直接返回 `false`，不产生任何意图。

- **相对定位拦截**：预估终点 $P_{start} + \Delta d$ 必须满足上述区间。若预判越界，指令立即拦截。

- **拦截优先级**：数值预检应早于状态机准入检查，确保非法指令在任何状态下都无法进入队列。

### 第三部分：动态方向互斥锁 (Directional Interlock)

当限位状态位（Bit）被触发时，系统进入“受限操作模式”。

- **定位指令全禁**：一旦触发任一方向限位，禁止所有 `MoveAbsolute` 和 `MoveRelative` 指令。
  - *逻辑理由*：限位属于非正常工况，强制要求人工通过点动（Jog）确认撤离方向。
- **点动定向拦截**：
  - **正限位触发时**：拦截 `Jog(Forward)`，仅允许 `Jog(Backward)`。
  - **负限位触发时**：拦截 `Jog(Backward)`，仅允许 `Jog(Forward)`。
- **运行中保护**：若在运动过程中监测到限位触发，且当前意图方向与限位方向一致，必须立即将 `m_pending_intent` 重置为 `monostate`（任务强迫终止）。

### 第四部分：安全逃逸语义 (Safety Recovery)

- **停止优先级**：限位状态下，`Stop` 指令必须始终保持最高穿透性。
- **自动解锁**：只有当反馈中的限位 Bit 恢复为 0 且坐标回归合法区间时，`Axis` 才会自动恢复全功能访问权限。

## 🧪 对应的 TDD 核心测试逻辑

| **测试组**     | **核心验证点**                                               |
| -------------- | ------------------------------------------------------------ |
| **数值镜像组** | 验证 `positiveSoftLimit()` 返回的值与 `applyFeedback` 中的输入一致。 |
| **数值拦截组** | 设定限位 1000，调用 `moveAbsolute(1001)` 必须返回 `false` 且不存入意图。 |
| **相对预判组** | 当前位置 900，限位 1000，调用 `moveRelative(150)` 必须被拦截。 |
| **正向逃逸组** | 在正限位 Bit=1 时，`jog(Forward)` 报错，`jog(Backward)` 成功产生意图。 |
| **运动中断组** | 正在 `Jog(Forward)` 过程中，反馈包突然出现 `posLimit = true`，意图必须瞬间消失。 |

## 🧠 架构师点评

通过这版抽象，你的 `Axis` 类实际上变成了一个**“地理围栏（Geo-fencing）”控制器**。

1. **对 HMI 友好**：你现在的 `Axis` 接口可以直接为界面提供 `posLimitValue`，并能通过 `move` 函数的返回值（false）直接给用户界面报错。
2. **逻辑下沉**：通过在适配层（Driver）读取 PLC 寄存器并填入 `AxisFeedback`，你成功地屏蔽了不同 PLC 品牌之间寄存器地址的差异，保持了 C++ 逻辑层的通用性。
