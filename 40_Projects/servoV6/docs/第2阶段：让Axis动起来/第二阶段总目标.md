## 🧠 标准写法（模板）

```cpp
class JogAxisUseCase {
public:
    JogAxisUseCase(IAxisDriver& driver)
        : driver_(driver) {}

    void execute(Axis& axis, Direction dir)
    {
        if (!axis.jog(dir)) {
            return;
        }

        driver_.send(axis.getPendingCommand());
    }

private:
    IAxisDriver& driver_;
};
```

------

## ❗核心原则

```text
✔ UseCase 只做流程
❌ 不写业务逻辑
❌ 不写寄存器
❌ 不改状态
```

## ✅ 接口定义

```cpp
};
```

------

## ✅ Fake 实现（先做这个！）

```cpp
class FakeAxisDriver : public IAxisDriver {
public:
    void send(const AxisCommand& cmd) override {
        history_.push_back(cmd);
    }

    const std::vector<AxisCommand>& history() const {
        return history_;
    }

private:
    std::vector<AxisCommand> history_;
};
```

------

## ❗为什么必须先 Fake？

```text
脱离 PLC 开发（你架构的核心优势）
```

# 🚨 六、你最容易踩的坑（提前帮你避掉）

------

## ❌ 坑1：在 UseCase 写 if/else 逻辑

👉 错误：

```cpp
if (axis.state() == Idle) { ... }
```

👉 正确：

```cpp
if (!axis.jog(dir)) return;
```

------

## ❌ 坑2：在 Driver 里判断状态

👉 错误：

```cpp
if (state == Idle) send();
```

👉 正确：

```text
Driver 永远执行命令
```

------

## ❌ 坑3：绕过 Axis 直接发命令

👉 绝对禁止：

```cpp
driver.send(JogCommand{});
```
