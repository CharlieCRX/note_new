# PLC 多端混合网络通讯配置与测试指南

## 1. 项目背景与目的

本指南旨在实现对单个底层 PLC 设备的“多端并发监控”。通过部署无线网桥（VAP11S）与工业交换机，构建一个同一网段下的混合局域网，使**有线 PC 端**与**无线安卓掌机端**能够同时通过 **Modbus TCP** 协议对 PLC 进行稳定的数据读写与控制。

## 2. 硬件清单与物理连接

### 2.1 硬件清单

* PLC 设备（带以太网口） × 1
* PC（上位机电脑） × 1
* 安卓掌机 × 1
* VAP11S 无线网桥/AP（及 12V 直流电源） × 1
* 工业以太网交换机（5口/8口）及电源 × 1
* 标准网线 × 3

### 2.2 物理接线拓扑

以**交换机**为核心节点进行星型连接：

1. **PLC 接入**：网线一端连接 PLC 以太网口，另一端接入交换机端口 1。
2. **网桥接入**：网线一端连接 VAP11S 网口，另一端接入交换机端口 2。
3. **PC 接入**：网线一端连接 PC 网口，另一端接入交换机端口 3。
4. **掌机接入**：安卓掌机通过 WiFi 无线连接至 VAP11S 发出的热点。

## 3. 网络 IP 规划表（核心规范）

为确保 Modbus TCP 顺利通讯且避免 IP 冲突，所有物理设备必须处于**同一网段**（前三段相同），且**主机号唯一**（最后一段不同）。

| 设备名称      | 接入方式    | 角色              | IP 地址         | 子网掩码        | 端口  |
| ------------- | ----------- | ----------------- | --------------- | --------------- | ----- |
| **PLC**       | 有线        | Modbus TCP Server | `192.168.1.111` | `255.255.255.0` | `502` |
| **PC 上位机** | 有线        | Modbus TCP Client | `192.168.1.88`  | `255.255.255.0` | -     |
| **安卓掌机**  | 无线 (WiFi) | Modbus TCP Client | `192.168.1.89`  | `255.255.255.0` | -     |

## 4. VAP11S 网桥配置步骤

1. **通电与登录**：为 VAP11S 供电，用电脑连接其网络，浏览器访问 `http://vonets.cfg` 或 `192.168.254.254`（默认账密均为 `admin`）。
2. **选择工作模式**：**必须选择“桥模式 (Bridge)”**。严禁选择“路由模式”，以确保无线终端与有线网络处于同一局域网。
3. **配置 WiFi 热点**：在“WiFi中继/热点设置”中，设置专属的 SSID（如 `PLC_Network`）及密码，**切勿勾选“禁用热点”**。
4. **保存生效**：提交设置并重启 VAP11S。

## 5. 终端设备静态 IP 设置

* **PC 端**：进入 Windows “网络和共享中心” -> 更改适配器设置 -> 以太网属性 -> IPv4，手动配置 IP 为 `192.168.1.88`。
* **安卓掌机端**：连接 `PLC_Network` WiFi 后，进入该 WiFi 的高级设置，将 IP 模式由“DHCP”改为“静态 (Static)”，手动配置 IP 为 `192.168.1.89`，网络前缀长度设为 `24`（即子网掩码 255.255.255.0）。

## 6. 联调与测试验收

测试应分为“网络底层”与“通讯协议层”两步进行：

### 第一步：网络连通性测试 (Ping)

1. **PC 端**：打开 CMD 命令提示符，输入 `ping 192.168.1.111`，确认有稳定毫秒级回复。
2. **掌机端**：使用 PingTools 等网络 APP，输入 `192.168.1.111` 执行 Ping 测试，确认无丢包（Timeout）。

### 第二步：Modbus TCP 协议交叉验证

1. **环境准备**：PC 端运行上位机软件或 Modbus Poll，掌机端运行 Modbus APP（目标 IP 均设为 `192.168.1.111`，端口 `502`）。
2. **读取测试**：两端同时对 PLC 的同一寄存器（如 40001 / MW0）发起 03 功能码（Read Holding Registers）读取请求，比对数值是否一致。
3. **写入验证**：
* 掌机端写入新数值（功能码 06/16），观察 PC 端监控界面是否同步更新。
* PC 端写入新数值，观察掌机端读取是否同步更新。
*(注：交叉验证全部通过，即代表网络与通讯架构搭建完全成功。)*



## 7. 后续应用开发注意事项

进入正式的软件/APP 编码阶段后，请注意以下工程细节：

1. **断线重连机制**：无线环境存在偶发波动，代码中必须实现 Socket 连接的超时检测（Timeout）与自动重连逻辑。
2. **通讯频率控制**：建议将客户端（特别是安卓端）的 Modbus 轮询频率（Polling Rate）设置在 **200ms ~ 500ms** 之间，避免给 PLC 网口造成并发拥堵。
3. **防止并发冲突**：PC端与掌机端同时具备控制权时，需在 PLC 程序内部或上位机业务逻辑中做好“互锁”机制，防止同一台设备的启动与停止指令被两端同时触发。
